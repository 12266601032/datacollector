Record Functions

Use record functions to get information about a record.

The expression language provides the following record functions:

Type. Syntax: ${record:type("<field-path>')}
Returns the data type of a field.
Possible return values are:
  • Boolean
  • Char
  • Byte
  • Short
  • Integer
  • Long
  • Float
  • Double
  • Date
  • Datetime
  • Decimal
  • String
  • Byte-Array
  • Map
  • List
${record:value('<field-path>')}
Returns the value of the field in the appropriate data type.
${record:id()}
Returns the ID of the record generated by a stage, generally a source, upon record creation.
${record:creator()}
Returns the stage name that created the record, generally a source.
${record:path()}
Returns the stages in the pipeline that the record passed through.

Error Functions

${error:stage()} The stage name that sent the record to error.

${error:code()} The error code used when sending the record to error.

${error:message()} the error message of the record

${error:collectorId()} The data collector ID that send the record to error (default value is hostname:port) ${error:pipeline()} The pipeline that sent the record to error

${error:time()} The time the record was sent to error

Basic Functions

${isIn(STRING full, STRING element)}: assumes 'full' is a list of comma separated values, looks for the element 'value'
