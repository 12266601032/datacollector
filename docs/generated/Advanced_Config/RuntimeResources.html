Runtime Resources

Similar to runtime properties, runtime resources are values that you define in a file local to the Data Collector and call from within a pipeline. But with runtime resources, you can restrict the permissions for the files to secure sensitive information. Use runtime properties to load sensitive information from files at runtime.

You can use runtime resources for any pipeline property that allows the use of the expression language. You can, for example, use runtime resources to represent user names and passwords, or OAuth authentication information.

To implement runtime resources, perform the following steps:
  1. Define each runtime resource.
  2. Use the runtime resource in the pipeline.

Defining Runtime Resources

Use the following steps to define a runtime resources:
  1. For each resource, create a text file and save it in the following directory:
    <SDCinstalldir>/resources

    A file must contain one piece of information to be used when the resource is called.

  2. Optionally, restrict the permissions for the file.

    Generally, anyone can read a file. To restrict permissions, configure the file so only the owner has read or write permissions for the file - in octals, that's 600 or 400. And the owner must be the system user that runs the Data Collector.

    When you use the resource in the pipeline, you specify whether the file is restricted.

Using Runtime Resources

Use the runtime:loadResource function to call a runtime resource. You can use runtime resources to represent sensitive information in any stage or pipeline property that allows the use of the expression language.

To call a runtime property, use the following syntax: 

runtime:loadResource("<file name>", <restricted: true | false>)
For example, the following expression returns the password for a JDBC Consumer origin. The file is named JDBCpassword.txt and is restricted so only the owner can read the file:
${runtime:loadResource("JDBCpassword.txt", true)
