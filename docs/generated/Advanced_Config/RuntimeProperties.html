Runtime Properties

Runtime properties are properties that you define in a file local to the Data Collector and call from within a pipeline. With runtime properties, you can define different sets of values for different Data Collectors.

You can use runtime properties to define values for stage and pipeline properties. For example, you can define an error directory runtime property that points to different directories on a test machine and a production machine. Similarly, you might create test and production runtime properties for the origin and destination stages.

You can use runtime properties for any pipeline property that allows the use of the expression language. You can, for example, use runtime properties to represent batch sizes and timeouts, directories, URI, and advanced properties. You cannot use runtime properties to represent fields or constants.

To implement runtime properties, perform the following steps:
  1. Define runtime properties.
  2. Use runtime properties in the pipeline.

Defining Runtime Properties

You can define runtime properties in the Data Collector configuration file or in a separate runtime properties file local to the Data Collector.

Data Collector configuration file
Use the following steps to define runtime properties in the Data Collector configuration file:
  1. In the Data Collector configuration file, configure the runtime.conf.location property as follows:
    runtime.conf.location=embedded
  2. Use the following format to define runtime properties: 
    runtime.conf_<property name>=<value>
    For example, the following runtime property defines a Hadoop FS directory template location: 
    runtime.conf_HDFSDirTemplate=/HDFS/DirectoryTemplate
Separate runtime properties file
Use the following steps to define runtime properties in a separate runtime properties file:
  1. Create a text file and save it in a directory local to the Data Collector.
  2. In the Data Collector configuration file, configure the runtime.conf.location property to point to the file.
  3. In the text file, list the runtime properties that you want to define using the following format:
    <property name>=<value>
    For example, the following runtime property defines the same Hadoop FS directory template in a separate runtime properties file:
    HDFSDirTemplate=/HDFS/DirectoryTemplate

Using Runtime Properties in the Pipeline

Use the runtime:conf function to call a runtime property. You can use runtime properties to represent any stage or pipeline property that allows the use of the expression language.

To call a runtime property, use the following syntax: 
runtime:conf('<property name>')
For example, to use a LogDir runtime property that represents the source file directory for the Directory stage, enter the following syntax for the Files Directory property:
${runtime:conf('LogDir')}
You can use a runtime property to represent a part of a property. For example, you could use a RootDir runtime property and append the rest of the directory in the property as follows:
${runtime:conf('RootDir')}/logfiles
