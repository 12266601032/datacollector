Installing and Launching the Data Collector (Service)

Install the Data Collector on the machine that you want to run a pipeline. After installation, you can launch the Data Collector.

Installing the Data Collector as a service requires sudo privileges on the root directory.

This procedure walks through setting the default directories, but you can alternatively use the libexec/sdcd-env.sh script.
  1. Download StreamSets SDC tarball and extract it to desired location.
  2. Copy initd/_sdcinitd_prototype to the /etc/init.d/ directory as sdc.
  3. Edit the sdc file and set the SDC_DIST environment variable to the location where you extracted the SDC tarball.
  4. Use the following command to make the sdc file executable: 
     sudo chmod 644 /etc/init.d/sdc
  5. Create a system user and group to run the Data Collector.
    The default user and group are sdc/sdc.
  6. Create the /etc/sdc directory and copy all files from etc into this directory.
  7. Create the SDC log directory at /var/log/sdc and change the owner to sdc:sdc.
  8. Create the SDC data directory at /var/run/sdc and change the owner to sdc:sdc.
  9. To launch the Data Collector, use the following command: 
    /etc/init.d/sdc start
