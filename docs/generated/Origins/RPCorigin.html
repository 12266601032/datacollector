RPC

The RPC origin enables connectivity between two StreamSets RPC pipelines. The RPC origin reads data passed from an RPC destination. Use the RPC origin as part of an RPC destination pipeline.

You can use RPC pipelines across a single Data Collector instance on a single machine, or across a local network or the public internet to a remote Data Collector. You can configure the origin to use SSL for secure connectivity.

When you configure the RPC origin, you define the port that the origin listens to for data. The port number must be specified in one of the RPC connections defined in the RPC destination.

Similarly, the RPC ID in the RPC origin must match the RPC ID used in the RPC destination.

Configuring an RPC Origin

Configure an RPC origin to process data from an RPC destination.

  1. In the Properties panel, on the General tab, configure the following properties:
    General Property Description
    Name Stage name.
    Description Optional description.
    On Record Error Error record handling for the stage:
    • Discard - Discards the record.
    • Send to Error - Sends the record to the pipeline for error handling.
    • Stop Pipeline - Stops the pipeline.
  2. On the RPC tab, configure the following properties:
    RPC Property Description
    RPC Listening Port Port number to listen to for data. Must match one of the port numbers associated with the RPC destination that provides the data.
    RPC ID User-defined ID. Must match the RPC ID defined in the RPC destination.
    Batch Wait Time (ms) Number of milliseconds to wait before sending a partial or empty batch.
    SSL Enabled Enables the secure transfer of data using SSL.

    To use SSL, both the RPC origin and RPC destination must be enabled for SSL.
    Keystore File Keystore file for SSL.

    Must be stored in the Data Collector resources directory: <SDCinstalldir>/resources.
    Keystore Password Password for the Keystore file.
  3. Optionally, click the Advanced tab to configure advanced properties:
    Advanced Property Description
    Max Record Size (MB) Maximum size for a record. Larger records are routed to the pipeline for error handling.

    Default is 10.
