Kafka Consumer

The Kafka Consumer reads data from a Kafka cluster and processes the data based on the type of data stored in Kafka. You can use the Kafka Consumer to read log, JSON, and CSV data from a Kafka cluster.

When you configure a Kafka Consumer, you configure the consumer group name, topic, and ZooKeeper connection information. You can optionally configure custom properties to add additional Kafka configuration properties.

When you start a pipeline, the Kafka Consumer becomes a new consumer group for the topic and starts reading data from the topic based on the offset provided by the ZooKeeper.

The Kafka Consumer processes data from all partitions in the topic. It saves the offset with the ZooKeeper after passing data to destinations.

Advanced Properties

You can add custom properties to pass Kafka configuration properties that you want to use.

When you add the Kafka configuration property, enter the Kafka property name and the value. The Kafka Consumer does not validate the property names or values.

Note: The Kafka Consumer uses the following Kafka configuration properties. If you add these properties, they do not override the values used by the Kafka Consumer:
  • auto.commit.enable
  • group.id
  • zookeeper.connect

Configuring a Kafka Consumer

Configure a Kafka Consumer to read log, JSON, CSV, or XML data from a Kafka cluster. When you configure the Kafka Consumer, you configure the general properties, including Kafka and ZooKeeper details. For JSON and CSV data, configure additional related properties. You can optionally add custom advanced properties.

  1. In the Properties panel, configure the following properties:
    General Properties Description
    Name Stage name.
    Description Optional description.
    Consumer Group Kakfa consumer group the Data Collector represents.
    Topic Kafka topic to read.
    ZooKeeper Connect String Connection string for the ZooKeeper of the Kafka cluster. To use more than one ZooKeeper, enter a comma-separated list.
    Max Batch Size Maximum number of messages processed at one time.
    Batch Duration Time (secs) Number of seconds the Kafka Consumer waits before sending a partial or empty batch.
    Payload Type Type of data in the topic. Use one of the following options:
    • Log
    • JSON
    • CSV
    • XML
  2. To configure additional Kafka properties, on the Advanced Properties tab, click Add and configure the following properties:
    Advanced Configuration Property Description
    Name Name of the Kafka configuration property. Enter the exact name.
    Value Value for the configuration option. Enter the value as expected by Kafka.
  3. For JSON data, click the JSON Data tab and configure the following properties:
    JSON Data Property Description
    JSON Content Type of content in JSON files. Use one of the following options:
    • Array of Objects
    • Multiple Objects.
    Maximum Object Length Maximum length for a JSON object. Longer objects are discarded.
  4. For CSV data, click the CSV Data tab and select the CSV Format for the data.
    Use one of the following options:
    • Basic CSV - File that includes comma-separated values.
    • MS Excel CSV - Microsoft Excel comma-separated file.
    • MySQL CSV - MySQL comma separated file.
    • RFC4180 CSV - Comma-separated file that strictly follows RFC4180 guidelines.
    • Tab-Separated Values - File that includes tab-separated values.
