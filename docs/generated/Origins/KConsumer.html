Kafka Consumer

The Kafka Consumer reads data from a Kafka cluster and processes the data based on the type of data stored in Kafka. You can use the Kafka Consumer to read log, delimited, JSON, and XML data from a Kafka cluster.

When you configure a Kafka Consumer, you configure the consumer group name, topic, and ZooKeeper connection information. You can add additional Kafka configuration properties as needed.

When you start a pipeline, the Kafka Consumer becomes a new consumer group for the topic and starts reading data from the topic based on the offset provided by the ZooKeeper.

The Kafka Consumer processes data from all partitions in the topic. It saves the offset with the ZooKeeper after passing data to destinations.

Additional Kafka Properties

You can add custom Kafka configuration properties to the Kafka Consumer.

When you add the Kafka configuration property, enter the Kafka property name and the value. The Kafka Consumer does not validate the property names or values.

Note: The Kafka Consumer uses the following Kafka configuration properties. The Kafka Consumer ignores user-defined values for these properties:
  • auto.commit.enable
  • group.id
  • zookeeper.connect

Configuring a Kafka Consumer

Configure a Kafka Consumer to read log, delimited, JSON, or XML data from a Kafka cluster. When you configure the Kafka Consumer, you configure the general properties, including Kafka and ZooKeeper details. For delimited and JSON data, configure additional related properties. You can optionally add custom Kafka properties.

  1. In the Properties panel, optionally enter a name and description for the stage.
  2. On the Kafka tab, configure the following properties:
    General Properties Description
    ZooKeeper Connection String Connection string for the ZooKeeper of the Kafka cluster. To use more than one ZooKeeper, enter a comma-separated list.
    Consumer Group Kakfa consumer group the Data Collector represents.
    Topic Kafka topic to read.
    Payload Type Type of data in the topic. Use one of the following options:
    • Log lines
    • JSON data
    • Delimited data
    • XML data
    Max Batch Size Maximum number of messages processed at one time.
    Batch Duration Time (secs) Number of seconds the Kafka Consumer waits before sending a partial or empty batch.
    Kafka Configuration Additional Kafka configuration properties to use. To add properties, click Add and define the Kafka property name and value.

    Use the property names and values as expected by Kafka.
  3. For JSON data, click the JSON Data tab and configure the following properties:
    JSON Property Description
    JSON Content Type of content in JSON files. Use one of the following options:
    • Array of Objects
    • Multiple Objects
    Produce Single Record Generates a single record for multiple JSON objects. Not available for JSON arrays.
    Max Object Length (chars) The maximum length of a JSON object.

    Longer objects are skipped and handled based on the pipeline error handling configuration.

  4. For CSV data, click the CSV Data tab and select the CSV Format for the data.
    Use one of the following options:
    • Basic CSV - File that includes comma-separated values.
    • MS Excel CSV - Microsoft Excel comma-separated file.
    • MySQL CSV - MySQL comma separated file.
    • RFC4180 CSV - Comma-separated file that strictly follows RFC4180 guidelines.
    • Tab-Separated Values - File that includes tab-separated values.
