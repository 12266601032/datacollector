Directory

The Directory origin reads data from files in a directory.

Directory reads the files in a directory with names that match a user-defined pattern. It can optionally begin processing with a specific file name. Then, it reads files with similar names in lexicographic ascending order. After processing a file, Directory can keep, archive, or delete the file.

Data Format

The Directory origin processes data differently based on the type of data in the file. Directory can process the following types of data:

Log
Generates a record for each line in the log file.
When you configure log data properties, you define the maximum number of characters for a line. When a line exceeds the configured number of characters, Directory truncates the line.
JSON
Generates a record for each JSON object.
You can use JSON files that include multiple JSON objects or a JSON array.
When you configure the JSON data properties, you define the maximum length of the JSON object. When an object exceeds the maximum length, Directory skips the object and continues processing with the next object.
The skipped object is either discarded or written to error based on the default error handling for the pipeline.
Delimited
Generates a record for each delimited record.
You can use the following types of delimited files:
  • Generic comma-separated values (CSV)
  • Microsoft Excel CSV
  • MySQL CSV
  • RFC4180 CSV
  • Tab-separated values
The Directory origin can convert a file to a map based on the header line of the file. If the file does not include a header, Directory creates the map based on column number.
XML
Generates records based on the location of the record separator XML element.
When you configure the XML data properties, you define the maximum length of the XML record.
When a record exceeds the maximum length, Directory skips the record and continues processing with the next record. The skipped record is either discarded or written to error based on the default error handling for the pipeline.

File Name Pattern

Use a regular expression to define the file name pattern. Directory reads files in the file directory that match the file name pattern.

For example, if you use *.json, Directory reads all files with the .json extension.

First File for Processing

Configure a first file for processing when you want Directory to ignore certain files in the file directory.

When you start a pipeline, Directory starts processing with the file name that you enter and continues processing files in naturally ascending order. If you did not configure a file name, it starts with the file with the lowest file name.

For example, the file directory includes the following files: File.log, File1.log, File2.log, File3.log. If you configure File1.log as the first file, Directory begins processes File1.log and continues with File2.log and File3.log. It does not process File.log.

File Wait Time and Empty Batches

When the Directory origin completes processing for existing files, it can send empty batches to pipeline destinations at regular intervals until new files appear. Use the File Wait Time general property to define the number of seconds to wait for new files before sending an empty batch.

Send empty batches when you want to write to destination systems regularly. For example, you might send empty batches to keep connections to destination systems from timing out or to trigger additional workflows in destinations systems.

Configuring a Directory Origin

Configure a Directory origin to read data from files in a directory. When you configure a Directory, you define general properties, including the data format to process. Then, you define post-processing options and the properties specific to the data format.

  1. In the Properties panel, optionally enter a name and description for the stage.
  2. On the Files tab, configure the following properties:
    File Property Description
    Name Stage name.
    Description Optional description.
    Data Format Data format for source files. Use one of the following formats:
    • Log data
    • JSON data
    • Delimited data
    • XML data
    Directory A directory local to the Data Collector where source files are stored.
    File Name Pattern Regular expression that describes the file names. For example, *.log.
    Max Files in Directory

    The maximum number of files that match the file name pattern that are allowed in the directory. When the directory contains more files, the pipeline fails.
    First File to Process Name of the first file to process. Use a name that uses the file name pattern.

    When you enter a file name, Directory reads the first file and other files in the directory with ascending names.
    File Wait Timeout (secs) Number of seconds to wait for a new file. If a new file does not arrive within the specified time, Directory sends an empty batch to destinations.
  3. On the Post Processing tab, configure the following properties:
    Post Processing Property Description
    Error Directory The directory for files that were not completely processed due to data handling errors.
    File Post-Processing The action taken after processing a file:
    • None - Keeps the file in place.
    • Archive - Moves the file to the archive directory.
    • Delete - Deletes the file.
    Archive Directory The directory for processed files. Available when you select Archive for the File Post Processing property.
    Archive Retention Time (mins) Number of minutes processed files are saved in the archive directory. Use 0 to keep archived files indefinitely.
  4. For log data, on the Log Data tab, configure the following properties:
    Log Data Property Description
    Maximum Line Length Maximum number of characters allowed for a line. Longer lines are truncated.
    Use Truncated Flag Adds a boolean field to the record to indicate if a record was truncated. The field name is Truncated. Possible values are True for a truncated record and False for untruncated values.
  5. For JSON data, on the JSON Data tab configure the following properties:
    JSON Data Property Description
    JSON Content Type of content in JSON files. Use one of the following options:
    • Array of Objects
    • Multiple Objects.
    Maximum Object Length Maximum length for a JSON object. Longer objects are skipped and handled based on the pipeline error handling configuration.
  6. For Delimited data, on the Delimited Data tab, configure the following properties:
    Delimited Data Property Description
    File Type File type. Use one of the following options:
    • Basic CSV - File that includes comma-separated values.
    • MS Excel CSV - Microsoft Excel comma-separated file.
    • MySQL CSV - MySQL comma separated file.
    • RFC4180 CSV - Comma-separated file that strictly follows RFC4180 guidelines.
    • Tab-Separated Values - File that includes tab-separated values.
    Header Line Indicates whether a file contains a header line.
    Convert to Map Converts a file to a map based on the fields in the header. If no header exists, uses column numbers to map the fields.
  7. For XML data, on the XML Data tab, configure the following properties:
    XML Data Property Description
    Element Record Delimiter XML element that acts as a record delimiter.
    Maximum Record Length Maximum length for a record. Longer records are discarded.
