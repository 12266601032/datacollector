Directory

The Directory origin reads data from log files, JSON files, and delimited files.

Directory reads the files in a directory with names that match a user-defined pattern. It can optionally begin processing with a specific file name. Then, it reads files with similar names in ascending order. After processing a file, Directory archives or deletes the file.

The Directory origin processes data differently based on the file type:

Log files
Generates a record for each line in the log file.
When you configure the Directory origin, you define the maximum number of characters for a line. When a line exceeds the configured number of characters, Directory truncates the line.
JSON files
Generates a record for each JSON object.
You can use JSON files that include multiple JSON objects or a JSON array.
When you configure the Directory origin, you define the maximum size of the JSON object. When an object exceeds the configured size, Directory skips the object and continues processing with the next object.
Delimited files
Generates a record for each delimited record.
You can use the following types of delimited files:
  • Generic comma-separated values (CSV)
  • Microsoft Excel CSV
  • MySQL CSV
  • RFC4180 CSV
  • Tab-separated values
The Directory origin can convert a file to a map based on the header line of the file. If the file does not include a header, Directory creates the map based on column number.

File Name Pattern

Use a regular expression to define the file name pattern. Directory reads files in the file directory that match the file name pattern.

For example, if you use *.json, Directory reads all files with the .json extension.

First File for Processing

Configure a first file for processing when you want Directory to ignore certain files in the file directory.

When you start a pipeline, Directory starts processing with the file name that you enter and continues processing files in ascending order. If you did not configure a file name, it starts with the lowest file in alphabetical order.

For example, the file directory includes the following files: File.log, File1.log, File2.log, File3.log. If you configure File1.log as the first file, Directory processes File1.log and continues with File2.log and File3.log. It does not process File.log.

File Wait Time and Empty Batches

When the Directory origin completes processing for existing files, it can send empty batches to pipeline destinations at regular intervals until new files appear. Use the File Wait Time general property to define the number of seconds to wait for new files before sending an empty batch.

Send empty batches when you want to write to destination systems regularly. For example, you might send empty batches to keep connections to destination systems from timing out or to trigger additional workflows in destinations systems.

Configuring Directory Properties

When you configure the Directory origin, configure the general properties, including the file type. Then, configure the properties based on the file type.

The Directory properties display on the following tabs in the Properties panel:

  • General. Configuration properties for all file types.
  • Log Files. Properties for log files.
  • JSON Files. Properties for JSON files.
  • Delimited Files. Properties for delimited files, such as comma-separated or tab-separated files.

General Properties

You can configure the following general properties for the Directory origin:

Directory General Property Description
Name Stage name.
Description Optional description.
Directory A directory local to the Data Collector where source files are stored.
File Name Pattern Regular expression that describes the file names. For example, *.log.
Max Files in Directory

The maximum number of files allowed in the directory. When the directory contains more files, the pipeline fails.
First File to Process Name of the first file to process. Use a name that uses the file name pattern.

When you enter a file name, Directory reads the first file and other files in the directory with ascending names.
File Post-Processing After processing a file, Directory performs one of the following tasks:
  • Archive - Moves the file to the archive directory.
  • Delete - Deletes the file.
Archive Directory The directory for processed files. Available when you select Archive for the File Post Processing property.
File Wait Time (secs) Number of seconds to wait for a new file. If a new file does not arrive within the specified time, Directory sends an empty batch to destinations.
Error Directory The directory for files that were not completely processed due to data handling errors.
Data Format Data format for source files. Use one of the following formats:
  • Log files
  • JSON files
  • Delimited files

Log File Properties

You can configure the following log file properties for the Directory origin:

Directory Log File Property Description
Maximum Line Length Maximum number of characters allowed for a line. Longer lines are truncated.

JSON File Properties

You can configure the following JSON file properties for the Directory origin:

Directory JSON File Property Description
JSON Content Type of content in JSON files. Use one of the following options:
  • Array of Objects
  • Multiple Objects.
Maximum Object Length Maximum length for a JSON object. Larger objects are not processed.

Delimited File Properties

You can configure the following delimited file properties for the Directory origin:

Directory Delimited File Property Description
File Type File type. Use one of the following options:
  • Basic CSV - file that includes comma-separated values
  • MS Excel CSV - Microsoft Excel comma-separated file
  • MySQL CSV - MySQL comma separated file
  • RFC4180 CSV - comma-separated file that strictly follows RFC4180 guidelines
  • Tab-Separated Values - file that includes tab-separated values
Header Line Indicates whether a file contains a header line.
Convert to Map Converts a file to a map based on the fields in the header. If no header exists, uses column numbers to map the fields.
