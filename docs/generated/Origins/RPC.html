RPC

The RPC origin reads data passed from an RPC destination over the HTTP or HTTPS. Use the RPC origin as part of an RPC pipeline that reads data from another RPC pipeline.

When you configure the RPC origin, you define the port that the origin listens to for data. You also define the RPC ID, batch wait time. You can also configure the origin to use HTTPS to securely transfer data.

  • Enabling SSL

    You can use HTTPS to securely transfer data to the RPC origin. To use HTTPS you need a valid keystore file and password.
