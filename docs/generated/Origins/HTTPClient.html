HTTP Client

The HTTP Client origin reads JSON data from an HTTP resource URL.

When you configure HTTP Client, you specify the resource URL and other read properties. When required, you can provide OAuth credentials or use runtime resources to save OAuth credentials in restricted files for runtime-only access.

Related concepts

Runtime Resources

Processing Mode

HTTP Client can use one of the following processing modes to read source data:
Streaming
HTTP Client maintains a connection and processes data as it becomes available. Use to process streaming data in real time.
Polling
HTTP Client polls the server at the specified interval for available data. Use to access data periodically, such as metrics and events at a REST endpoint.

HTTP Method

To request data from an HTTP resource URL, specify the request method to use. Most servers require a GET request, but you should verify the request required by the server you want to access.

You can use the following methods:
  • GET
  • PUT
  • POST
  • DELETE
  • HEAD

Configuring an HTTP Client Origin

Configure an HTTP Client origin to read JSON data from an HTTP resource URL.

  1. On the HTTP tab, configure the following properties:
    HTTP Property Description
    Data Format HTTP Client reads JSON data.
    Resource URL URL where the data resides.
    HTTP Method HTTP method to use to request data from the server.
    Request Timeout Milliseconds to wait for the server to begin sending a response.
    Mode Processing mode:
    • Streaming - Maintains a connection and processes data as it becomes available.
    • Polling - Connects periodically to check for data.
    Polling Interval (ms) Milliseconds the origin waits before checking for new data. Used in the polling mode only.
    Use OAuth Determines whether OAuth credentials are used to connect to the server.
    Batch Size Maximum number of JSON objects to include in a batch and send through the pipeline at one time.
    Batch Wait Time (ms) Maximum amount of time to wait before sending a partial or empty batch.
  2. When using OAuth, on the OAuth tab, configure the following properties:
    OAuth Property Description
    Consumer Key OAuth consumer key.
    Consumer Secret OAuth consumer secret.
    Token OAuth consumer token.
    Token Secret OAuth token secret.
  3. On the JSON tab, configure the following properties:
    JSON Property Description
    JSON Content Type of JSON content to be processed:
    • JSON Array of Objects - Use to process data in polling mode.
    • Multiple JSON Objects - Use to process data in streaming or polling mode.
    Entity Delimiter Delimiter to indicate the end of a JSON record, such as \n or \r. Used when processing multiple JSON objects.
