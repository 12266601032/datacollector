File Processing

Directory processes files in a directory that match the configured file name pattern.

When you define a first file to process, it starts processing with the specified file and continues processing files in naturally ascending order. When you do not specify a first file, Directory processes all files in the directory with that match the file name pattern.

Configure a first file for processing when you want Directory to ignore older files in the directory.

For example, the file directory includes the following files: File.log, File1.log, File2.log, File3.log. If you configure File2.log as the first file, Directory begins with that file and continues to File3.log. It does not process the earlier files.
