Origins

An origin stage represents the data that you want the pipeline to process. You can use a single origin stage in a pipeline.

You can use the following types of origins in a pipeline:
  • Directory - Reads files from a directory.
  • File Tail - Reads data as it is written to a file.
  • Kafka Consumer - Reads data from Kafka

Previewing Raw Source Data

You can preview raw source data when you configure an origin. Preview raw source data to verify the correct configuration for the origin.

When you request a preview a file origin, you enter a directory and file name. You can use the real directory and actual source file. Or when appropriate, you might use a different file that is similar to the source. When you preview a Kafka Consumer, you enter the connection information for the Kafka cluster.

The data used for the raw source preview in an origin stage is not used when previewing data for the pipeline.

  1. In the origin stage, in the Raw Preview tab.
  2. For file origins, enter a directory and file name. For the Kafka Consumer, enter the following information:
    Kafka Raw Preview Property Description
    Topic
    Partition  
    Broker Host  
    Broker Port  
    Batch Duration Time  
  3. Click Preview.
The Raw Source Preview area displays the preview.
