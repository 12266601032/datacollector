File Tail

The File Tail origin reads data as it is written to a file. As long as the pipeline runs, File Tail captures log or JSON data that is written to the file. The File Tail origin does not read data already in the file or data that is written to the file when the pipeline is not running.

For example, you have a log file with 30 rows of data when you start the pipeline. The File Tail origin ignores the existing data and begins processing with the 31st row of data when it arrives. It continues capturing incoming data until you stop the pipeline.

When you restart the pipeline, the File Tail origin begins processing with the next row of incoming data. It does not try to process any rows that were written to the log file while the pipeline was stopped.

Batch Size and Wait Time

For the File Tail origin, the batch size determines the maximum number of rows sent through the pipeline at one time. The batch wait time determines the time that File Tail waits for additional data before sending a batch. At the end of the wait time, it sends the batch regardless of how many rows the batch contains.

For example, say the batch size is 20 rows and the batch wait time is 240 seconds. When data arrives quickly, File Tail fills a batch with 20 rows and sends it through the pipeline immediately, creating a new batch and sending it again as soon as it is full. As incoming data slows, a remaining batch contains a few rows, gaining an extra row periodically. 240 seconds after creating the batch, File Tail sends the partially-full batch through the pipeline. It immediately creates a new batch and starts a new countdown.

Configure the batch wait time based on your processing needs. You might reduce the batch wait time to ensure all data is processed within a specified time frame or to make regular contact with pipeline destinations. Use the default or increase the wait time if you prefer not to process partial or empty batches.

Configuring a File Tail Origin

Configure a File Tail origin to read data as it is written to a file. You can read JSON or log data.

  1. In the Properties panel, optionally enter a name and description for the stage.
  2. On the File tab, configure the following properties:
    File Tail General Property Description
    Data Format Format of data in the file. Use one of the following options:
    • Log data
    • JSON data
    File Path Directory and file name to read.
    Batch Size (records) Maximum number of records to include in a batch.
    Batch Wait Time (secs) Number of seconds to wait before sending a batch.
