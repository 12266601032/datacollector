File Tail

The File Tail origin reads data as it is written to a file. As long as the pipeline runs, File Tail captures text or JSON data that is written to the file. The File Tail origin does not read data already in the file or data that is written to the file when the pipeline is not running.

For example, you have a log file with 30 rows of data when you start the pipeline. The File Tail origin ignores the existing data and begins processing with the 31st row of data when it arrives. It continues capturing incoming data until you stop the pipeline.

When you restart the pipeline, the File Tail origin begins processing with the next row of incoming data. It does not try to process any rows that were written to the log file while the pipeline was stopped.
Note: You can use data preview to test stage and pipeline behavior, but only when data is written to the file at the time of preview.

Tailing JSON Data

To capture JSON data with the File Tail origin, each JSON object must be written to file as a single line.

Configuring a File Tail Origin

Configure a File Tail origin to read data as it is written to a file. You can read text or JSON data.

  1. In the Properties panel, optionally enter a name and description for the stage.
  2. On the File tab, configure the following properties:
    File Property Description
    Data Format Format of data in the file. Use one of the following options:
    • Text
    • JSON
    File Path Directory and name of the file to read.
    Maximum Lines per Batch (recs) Maximum number of records to include in a batch.
    Batch Wait Time (secs) Number of seconds to wait before sending a batch.
