Metric Rules and Alerts

Metric rules provide feedback on pipeline and stage performance, such as batch processing times.

When you enable a metric rule, it automatically enables an alert for the rule. Metric alerts display in the Data Collector console in monitor mode. You can also configure a metric rule to send email alerts to all email addresses associated with the pipeline.

You can configure metric rules when you configure or monitor the pipeline. Metric rules take effect after you enable them.

The Data Collector provides a set of metric rules for all pipelines. By default, they are not enabled. You can edit, delete, and enable the rules. You can also create custom metric rules.

Metric Types

You can use different metric types when you create a metric rule. The metric type determines the type of alert that you can create. For example, when you select a counter type, you can create alerts based on the number of error records generated by a specific stage.

You can use the following metric types in metric rules:

Gauge
Provides alerts based on the age of the current batch, the amount of time a stage takes to process a batch, or the time the Data Collector last received a record from the origin.
Counter
Provides alerts based on a count of different record types and of stage errors for any stage in the pipeline.
Histogram
Provides alerts based on a histogram of different record types and stage errors for any stage. You can also use batch-based histograms.
Meter
Provides alerts based on a rates of different record types and stage errors for any stage. You can also use batch-based meters.
Timer
Provides alerts based on batch processing timers for the pipeline or any stage in the pipeline.

Configuring a Metric Rule and Alert

Create a metric rule to receive alerts when the specified conditions occur. You can configure metric rules and alerts when you configure or monitor a pipeline. You can edit or delete metric rules when they are not enabled.

Configure metric alerts based on the metric type, the metric, an element associated with the metric, and a condition.
  1. In the Properties panel or Monitor panel, click the Metric Rules tab, and click the Add icon.
  2. In the Metric Rule dialog box, configure the following properties:
    Metric Rule Property Description
    Alert Text Text to display when the alert is triggered.

    Enter text that explains the reason for the alert. For example, "Over 1000 pipeline error records."
    Metric Type Type of metric information the alert is based on:
    • Gauge
    • Counter
    • Histogram
    • Meter
    • Timer
    Metric ID Metric to use. Provides a list of available metrics based on the metric type.
    Metric Element Metric element to use. Provides a list of available elements based on the metric ID.
    Condition Condition to trigger the alert. Use the expression language to configure the condition.
    Send Email Sends an email when the alert is triggered.

    To send an email, add email addresses for alerts and configure the Data Collector email properties.
  3. Click Save.
    The new metric rule displays in the list.
  4. To enable the rule, select Active.
    You can enable and disable email alerts from the list of rules. You can edit and delete inactive rules.
