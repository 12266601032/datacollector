Command Line Interface

The Data Collector provides a command line interface.

You can use the following commands with the command line interface. Use the help command for details about each command.
definitions
Provides the following command:
  • list - Lists definitions of a pipeline and the stages available for use.
help
Provides information about each command.
manager
Provides the following commands:
  • alerts - Returns all triggered alerts.
  • delete-alerts - Clears triggered alerts.
  • delete-history - Deletes the pipeline history.
  • error-messages -
  • error-records - Returns error records for the pipeline. You can request error records generated by a specific stage, records under a certain size.
  • history - Returns the history of a pipeline.
  • metrics - Returns pipeline metric information.
  • reset-origin - Resets the origin when possible.
  • sampled-records - Returns the records sampled for the pipeline.
  • snapshot-capture - Captures a snapshot.
  • snapshot-data - Returns the data captured for a snapshot.
  • snapshot-delete - Deletes a snapshot.
  • snapshot-list - Lists all snapshot information.
  • snapshot-status - Returns the status of a snapshot.
  • start - Starts a pipeline.
  • status - Returns the status of a pipeline.
  • stop - Stops a pipeline.
ping
Pings the Data Collector.
preview
Provides the following preview commands:
  • data - Returns preview data.
  • run - Starts data preview.
  • status - Returns the preview/validation status.
  • stop - Stops the data preview.
  • validate-pipeline - Performs explicit validation for a pipeline.
store
Provides the following commands:
  • create - Creates a pipeline.
  • delete - Deletes a pipeline.
  • export - Exports a pipeline.
  • get-config - Returns pipeline configuration information.
  • get-rules - Returns pipeline rules.
  • import - Imports a pipeline.
  • list - Lists information for all available pipelines.
  • update-config - Updates pipeline configuration.
  • update-rules - Imports pipeline rules defined in a separate file to the pipeline. The file must include pipeline rules defined in JSON.
system
Provides the following system commands:
  • configuration - Returns properties defined in the Data Collector properties file.
  • current-user - Returns information about the user logged in to the command line.
  • directories - Returns the Data Collector directories.
  • info - Returns build information for the Data Collector.
  • server-time - Returns the server time.
  • shutdown - Shuts down the Data Collector.
  • threads - Returns a list of active threads, a.k.a, a thread dump.

Using the Command Line Interface

Use the following command to launch the Data Collector command line interface:
$ <SDCinstalldir/bin/streamsets cli (-U <sdcURL> | --url <sdcURL>) \
[(-a <sdcAuthType> | --auth-type <sdcAuthType>)] [(-u <sdcUser> | --user <sdcUser>)] \
[(-p <sdcPassword> | --password <sdcPassword>)] <command> [<args>]
The following table describes the parameters in the command:
Parameter Description
<sdcURL> Data Collector URL.

The default URL is http://localhost:18630/.
<sdcAuthType> Optional. HTTP authentication type used by the Data Collector.
<sdcUser> Optional. User name for the login.
<sdcPassword> Optional. Password for the login.
<command> <args> Required. Command to perform.
For additional information for each command, including the arguments you can use, use the help command as follows: 
<SDCinstalldir>/bin/streamsets cli (-U <sdcURL> | --url <sdcURL>) \
[(-a <sdcAuthType> | --auth-type <sdcAuthType>)] [(-u <sdcUser> | --user <sdcUser>)] \
[(-p <sdcPassword> | --password <sdcPassword>)] help <command>
For example, the following command displays the details for the manager command: 
<SDCinstalldir>/bin/streamsets cli -U http://localhost:18630 help manager
