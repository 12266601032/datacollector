Flume

The Flume destination writes data to Flume. When you write data to Flume, you pass data to a Flume client. The Flume client passes data to Flume hosts based on Flume client configuration properties.

When you configure a Flume destination, you can configure one or more Flume hosts to use, connection information, the Flume client type, and the data format to use. You can also configure some Flume client properties, such as the Flume batch size or connection timeout.

The Flume destination can write data to the following Flume client types:
  • Apache Avro Failover RPC Client
  • Apache Avro LoadBalancing RPC Client
  • Apache Thrift

Data Formats

The Flume destination writes data to Flume based on the data format that you select. You can use the following data formats:
Text
The destination writes a single text field of a record. When you configure the stage, you select the field to use. When necessary, merge record data into the field earlier in the pipeline.
JSON
The destination writes records as JSON data. You can use one of the following formats:
  • Array - Each file includes a single array. In the array, each element is a JSON representation of each record.
  • Multiple objects - Each file includes multiple JSON objects. Each object is a JSON representation of a record.
Delimited
The destination writes records as delimited data. When you use this data format, you must ensure that the data uses the following data structure:
  • A root field that contains an array of maps.
  • Each map includes a value element and an optional header element.
SDC Record
The destination writes records in the SDC Record data format.

Configuring a Flume Destination

Configure a Flume destination to write data to Flume.

  1. In the Properties panel, on the General tab, configure the following properties:
    General Property Description
    Name Stage name.
    Description Optional description.
    Required Fields Fields that must include data to be passed to the stage.
    Tip: You might include fields that the stage uses.
    On Record Error Error record handling for the stage:
    • Discard - Discards the record.
    • Send to Error - Sends the record to the pipeline for error handling.
    • Stop Pipeline - Stops the pipeline.
  2. On the Flume tab, configure the following properties:
    Flume Property Description
    Host Configuration Flume host information:
    • Name - Local alias for the Flume host.
    • Flume host and port - Host name and port number for the Flume agent. Use the following format: <host name>:<port number>
    Click Add to add additional host configuration.
    Client Type Flume source client type:
    • Avro Failover - Apache Avro Failover RPC Client.
    • Avro Load Balancing - Apache Avro LoadBalancing RPC Client.
    • Thrift - Apache Thrift RPC Client.
    Backoff The Avro load balancing client temporarily avoids writing to a failed host.
    Max Backoff (ms) The maximum amount of time that the Avro load balancing client avoids writing to a failed host.

    Enter the number of milliseconds between 0 and 65536.

    Use 0 to trigger the Flume default value of 30000 milliseconds - 30 seconds.

    Host Selection Strategy

    Strategy the Avro load balancing client uses to select a Flume host.
    Flume Batch Size (events) Maximum amount of events that the Flume client writes to a Flume host at one time.

    The Flume client can split a Data Collector batch into multiple Flume batches before passing Flume batches to the Flume host.
    Flume Client Connection Timeout (ms) Maximum amount of time for the Flume client to connect to the Flume host.
    Flume Client Request Timeout (ms) Milliseconds the Flume client waits for a response after sending data to the Flume host.

    When the client does not receive a response in time, the Flume destination resends the batch. It resends up to the number configured for Max Retry Attempts.
    Max Retry Attempts Maximum number of times the Flume destination resends data to Flume and waits for a response before stopping the pipeline.
    Retry Wait Time (ms) Milliseconds to wait before resending data to Flume.
    Data Format Data format for output events:
    • Text
    • JSON
    • Delimited
    • SDC Record
    Event Charset Character encoding of the events to be processed.
    One Event per Batch

    Generates a single event for all records in the batch.
