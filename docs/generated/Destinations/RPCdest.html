RPC

The RPC destination passes data over the WAN to an RPC origin. Use the RPC destination as part of an RPC origin pipeline.

When you configure the RPC destination, you specify the RPC ID and RPC connections to use. You can optiionally enable SSL to pass data securely and define retry and timeout properties.

Related concepts

RPC Pipelines

RPC Connections

When you configure an RPC destination, you configure RPC connections to define where to pass data.

The RPC connection is the host and port number of the RPC origin to receive the data. You can pass data to multiple RPC origins to allow processing of data by multiple destination pipelines.

Enabling SSL

You can configure the RPC destination to enable the secure transfer of data with SSL. To enable SSL, you need a truststore, or trusted keystore, that includes the public key of the SSL certificate of the RPC origin, and the associated password.

To enable security:
  1. Save the truststore file in the Data Collector resources directory: <SDCinstalldir>/resources.
  2. In the RPC destination, enter the filename and password.

Configuring an RPC Destination

Configure an RPC destination to pass data to an RPC origin in an RPC destination pipeline.
  1. In the Properties panel, on the General tab, configure the following properties:
    General Property Description
    Name Stage name.
    Description Optional description.
    On Record Error Error record handling for the stage:
    • Discard - Discards the record.
    • Send to Error - Sends the record to the pipeline for error handling.
    • Stop Pipeline - Stops the pipeline. Not valid for cluster pipelines.
    Required Fields Fields that must include data to be passed into the stage.
    Tip: You might include fields that the stage uses.
    Preconditions Conditions that must evaluate to TRUE to allow a record to enter the stage for processing. Click Add to create additional preconditions.
  2. On the RPC tab, configure the following properties:
    RPC Property Description
    RPC Connection Connection information for the RPC destination pipeline to continue processing data. Use the following format: <host>:<port>.

    Use a single RPC connection for each destination pipeline. Add additional connections as needed.

    Use the port number when you configure the RPC origin in the destination pipeline.
    RPC ID User-defined ID to allow the destination to pass data to an RPC origin. Use this ID in all RPC origins to process data from the destination.
    SSL Enabled Enables the secure transfer of data over the WAN using SSL.
