Destinations

  • Destinations

    A destination stage represents the target data for the pipeline. You can use one or more destinations in a pipeline.
  • Required Fields

    You can configure required fields for any processor or target stage. A required field must exist in a record to keep it in the pipeline.
  • Hadoop FS

    The Hadoop FS destination writes data to a Hadoop file system. You can write delimited or JSON data to flat files or Hadoop sequence files.
  • Kafka Producer

    The Kafka Producer passes data to a Kafka cluster. The Kakfa Producer can pass data to Kafka as delimited, text, or JSON data.
  • Local File System

    Use the Local File System destination to write bad records from the pipeline to files in a local file system.
  • Trash

    The Trash destination discards all records. Use the Trash destination as a visual representation of records dropped from the pipeline. Or, use the Trash destination during development as a placeholder for another destination.
