Kafka Producer

The Kafka Producer passes data to a Kafka cluster. The Kakfa Producer can pass data to Kafka as delimited, text, or JSON data.

When you configure a Kafka Producer, you configure the brokers, topic, partition strategy, and data format to use. You can add additional Kafka configuration properties as needed.

The Kafka Producer passes data to partitions in the Kafka topic based on the partition strategy that you choose.

Writing Text and Delimited Data

When you write delimited data to Kafka, you define the fields to pass data to the cluster. When you write text data, you define a single field to pass the data.

When necessary, use an Expression Evaluator to merge data into the fields that you want to use.

Broker List

The Kafka Producer connects to Kafka based on the topic and associated brokers that you specify. To ensure a connection in case a specified broker goes down, list as many brokers as possible.

Additional Kafka Properties

You can add custom Kafka configuration properties to the Kafka Producer.

When you add the Kafka configuration property, enter the Kafka property name and the value. The Kafka Consumer does not validate the property names or values.

Note: The Kafka Producer uses the broker.host Kafka configuration property. The Kafka Consumer ignores user-defined values for the broker.list property.

Configuring a Kafka Producer

Configure a Kafka Producer to write text, delimited, or JSON text to a Kafka cluster.

  1. In the Properties panel, optionally enter a name and description for the stage.
  2. For Required Fields, enter any fields that you want to require data.
    You might include any field that the processor uses.
  3. On the Kafka tab, configure the following properties:
    Kafka Properties Description
    Broker URIs A comma-separated list of URIs for Kafka brokers that write to the topic. Use the format <host>:<port>.

    To ensure a connection, enter as many broker URIs as possible.
    Topic Kafka topic to use.
    Partition Strategy Strategy to use to write to partitions:
    • Round Robin - Takes turns writing to different partitions.
    • Random - Writes to partitions randomly.
    • Expression - Uses an expression to evalulate when to write to different partitinos.
    Expression Expression to use to write to parititons with an expression.
    Data Format Data format for output messages:
    • Text
    • JSON
    • Delimited
    Kafka Configuration Additional Kafka properties to use. Click the Add icon and define the Kafka property name and value.

    Use the property names and values as expected by Kafka. Do not use the broker.list property.
  4. For delimited data, click the Delimited tab and configure the following properties:
    Delimited Property Description
    Delimiter Format Format to use for delimited data:
    • Basic CSV - File that includes comma-separated values.
    • MS Excel CSV - Microsoft Excel comma-separated file.
    • MySQL CSV - MySQL comma separated file.
    • RFC4180 CSV - Comma-separated file that strictly follows RFC4180 guidelines.
    • Tab-Separated Values - File that includes tab-separated values.
    Fields Fields to include in the record.
  5. For log or text data, click the Text tab and configure the following property:
    Text Property Description
    Field Single field to write data to Kafka.
