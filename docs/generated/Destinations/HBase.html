HBase

The HBase destination writes data to an HBase cluster. The destination writes data to HBase in text, binary, or JSON format. You can define the data format for each column written to HBase.

When you configure the HBase destination, you specify the HBase configuration properties, including the ZooKeeper Quorum, parent znode, and table name. You specify the row key for the table, and then map fields from the pipeline to HBase columns.

When necessary, you can configure Kerberos authentication and add additional HBase configuration properties.

Additional HBase Properties

You can add custom HBase configuration properties to the HBase destination.

When you add an HBase configuration property, enter the exact property name and the value. The HBase destination does not validate the property names or values.

Configuring an HBase Destination

Configure an HBase destination to write data to HBase.

  1. In the Properties panel, on the General tab, configure the following properties:
    General Property Description
    Name Stage name.
    Description Optional description.
    Stage Library Library version that you want to use.
    Required Fields Fields that must include data to be passed to the stage.
    Tip: You might include fields that the stage uses.
    On Record Error Error record handling for the stage:
    • Discard - Discards the record.
    • Send to Error - Sends the record to the pipeline for error handling.
    • Stop Pipeline - Stops the pipeline.
  2. On the HBase tab, configure the following properties:
    HBase Property Description
    ZooKeeper Quorum Comma-separated list of servers in the ZooKeeper quorum. Use the following format: <host>.<domain>.com.

    To ensure a connection, enter additional broker URIs.
    ZooKeeper Client Port Port number clients use to connect to the ZooKeeper servers.
    ZooKeeper Parent Znode Root node that contains all znodes used by the HBase cluster.
    Table Name Name of the HBase table to use. Enter a table name or a namespace and table name as follows: <namespace>.<tablename>.

    If you do not enter a table name, HBase uses the default namespace.

    Row Key Row key for the table.
    Storage Type Storage type of the row key. Choose Text or Binary.
    Field to Column Mapping Map fields from pipeline records to HBase columns, and then define the storage type for the column in HBase.

    Use the Add icon to create additional field mappings.

    Kerberos Authentication Indicates that HBase cluster uses Kerberos authentication.
    Kerberos Principal

    Kerberos principal used to connect to HBase.

    Kerberos Keytab

    Location for the keytab file that contains the credentials for the Kerberos principal.

    HBase Configuration

    Additional HBase configuration properties to use.

    To add properties, click Add and define the HBase property name and value. Use the property names and values as expected by HBase.
