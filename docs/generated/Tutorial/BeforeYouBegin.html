Before You Begin

Before you start this tutorial, you'll need a few things:

  1. Download sample data - You can download sample data for this tutorial from the StreamSets website:

    To run the pipeline, you'll need the larger file to provide enough data to see how things work. If you don't plan to run the pipeline, download the smaller file so you can at least preview data and examine how the pipeline processes a sample set.

  2. Set up origin and destination and error directories:
    • Origin directory - Save the sample data in a directory local to the Data Collector. Ensure that the user who started the Data Collector has permission to read from the directory.
    • Destination directory - The pipeline writes output records to file. Consider where you want to write the output files and make sure the user has write permission for the directory.
    • Error directory - The pipeline writes error records to file. Consider where you want to write the error files and make sure the user has write permission for the directory.
  3. The Data Collector should be installed and running, but not running other pipelines. At this time, a Data Collector instance can run a single pipeline, but you can install multiple instances on the same machine.
  4. To create and run the pipeline, you should have a login with the creator and manager roles or the admin role. If using the default logins shipped with the Data Collector, you can use the admin login: admin / admin.
