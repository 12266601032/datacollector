Before You Begin

Before you start this tutorial, you'll need to do a few things:

  1. Download sample data.

    You can download sample data from the StreamSets website. You'll need a login to the password-protected section of the site - it just takes a moment to set up an account.

    Sample data URL:

  2. Set up directories.

    We will use prefixes for output and error file names, so you can use the same directory for all files as long as the user who started the Data Collector has read and write permission for the directory.

    If you prefer to use separate directories, set up the following:

    • Origin directory - Save the sample data in a directory local to the Data Collector. Make sure that the user who started the Data Collector has permission to read from the directory.
    • Destination directory - The pipeline writes output records to file. Consider where you want to write the output files and make sure the Data Collector user has write permission for the directory.
    • Error directory - The pipeline also writes error records to file - though you can configure it to write to Kafka if you prefer. Make sure the Data Collector user has write permission for the error directory.
  3. The Data Collector should be installed and running.
  4. To create and run the pipeline, you should have a login with admin role or both the creator and manager roles.

    If using the default accounts shipped with the Data Collector, you can use the admin login: admin / admin.
