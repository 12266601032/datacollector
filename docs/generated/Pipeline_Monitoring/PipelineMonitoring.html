Pipeline Monitoring

When the Data Collector runs a pipeline, you can view real-time statistics about the pipeline, examine a sampling of the data being processed, and create rules and alerts.

When you access the Data Collector console as the Data Collector runs a pipeline, the console displays the pipeline in Monitoring mode. In Monitoring mode, you can perform the following tasks:
  • View how a snapshot of data moves through the pipeline
  • View statistics for the pipeline and stages in the pipeline
  • Create metric alerts based on pipeline and stage statistics
  • Create data rules that can generate graphs and alerts based on a sampling of data passing between stages
  • Define the email addresses to receive email alerts
  • View the pipeline history.

Data Collector Console - Monitor Mode

In Monitor mode, you can use the Data Collector console to view data as it passes through the pipeline.

Summary

Displays the record count for the pipeline, record and batch throughput for the pipeline or individual stages, and batch processing statistics for the pipeline or individual stages.

Tip: You can hover over different parts of the charts to view exact numbers.
Error For a pipeline, displays histograms for the number of error records by five minute decay and the number of error records by stage since the pipeline started.

For a stage, displays the number of error records and the number of stage errors.

Configuration Displays configuration information for the pipeline or selected stage. You cannot configure the pipeline when it is running.
History Displays the status and history of the pipeline.
