Required Fields

You can configure required fields for any processor or target stage. A required field must exist in a record to keep it in the pipeline.

When a record does not include a required field, the stage either discards the record or writes it to error based on the default pipeline error handling.

Configure required fields based on your business requirements or in order to minimize processing errors.

For example, a Field Hasher encodes social security data in the /SS field. To ensure processing only records with social security numbers, make /SS a required field.
