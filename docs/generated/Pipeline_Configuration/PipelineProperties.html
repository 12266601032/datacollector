Pipeline Properties

Pipeline properties apply to the entire pipeline. Pipeline properties display in the Properties panel when you click on the pipeline canvas.

Delivery Guarantee

When you configure a pipeline, you define how you want data to be treated: Do you want to prevent the loss of data or the duplication of data?

The Delivery Guarantee pipeline property offers the following choices:
At least once
The Data Collector ensures that the pipeline processes all data.
If a failure causes the Data Collector to stop while processing a batch of data, when restarts, it reprocesses the batch.
This option ensures that no data is lost. But if a failure occurs as the Data Collector writes data to destinations, up to one batch data might be duplicated in the target.
At most once
The Data Collector ensures that data is not processed more than once.
If a failure causes the Data Collector to stop while processing a batch of data, when it starts up, it begins processing with the next batch of data.
This option ensures duplicate data is not written to the target due to reprocessing. But if a failure occurs as the Data Collector writes data to destinations, up to one batch of data might not be processed.

Pipeline Memory

The Data Collector uses memory when it runs a pipeline. To avoid causing out-of-memory errors on the host machine, you can configure the maximum amount of memory that can be used for the pipeline.

Best practice is to configure the pipeline to use up to 65% of the Java heap size defined for the Data Collector. For example, with the default Data Collector Java heap size of 1024 MB, the highest setting for the pipeline memory should be 665 MB.

Use the Max Pipeline Memory pipeline property to define the maximum amount of memory to use. When you configure the pipeline, you can also configure the action to take when pipeline memory usage exceeds the configured amount.

Tip: To avoid errors, you might create an alert to indicate when the memory usage approaches the configured limit.

Pipeline Constants

A pipeline constant is a constant that you define for the pipeline and that you can use in any stage in the pipeline. Define a pipeline constant when you have a constant that you want to update easily or to use more than once.

For example, if you have a bonus percentage that you update quarterly, you might create a Bonus constant so you can update the value easily in the pipeline properties and have the value propagated throughout the pipeline. Or, if you use a common root directory that you use for source files and error files, you might create a Root constant that you use as a basis for both directories, e.g. Root/log and Root/error.

You can enter any text as the value for a pipeline constant. When the pipeline runs, the value replaces the name of the pipeline constant.
