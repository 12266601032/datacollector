Retrying the Pipeline

By default, when the Data Collector encounters a stage-level error that might cause a pipeline to fail, it retries the pipeline - waiting a period of time, and then trying again to run the pipeline.

A stage-level error might include a destination not being able to write to the destination system because the system is down.

By default, the Data Collector retries the pipeline an infinite number of times. This allows the pipeline to continue running as soon as any temporary connection or system failures resolve.

You can use the Retry Attempts pipeline property to define the number of pipeline retries that the Data Collector attempts. -1 represents an infinite number of retries.

The wait time between retries begins at 15 seconds and doubles in length until reaching a maximum of 5 minutes.
