Error Handling

Each stage in the pipeline includes built-in resilience. Some stages also include configurable error handling. When the Data Collector encounters an unexpected error, it stops the pipeline.

When a stage encounters a record that it does not expect, it discards the record. For example, you have a Directory origin configured to read JSON objects with a maximum length of 1000. When the Directory origin encounters a JSON object with a length of 1001 or longer, it drops the object and continues with the next JSON object. As another example, if you configure a processor stage to require an ID field, it drops any record that does not include the field.

Several processors, including the Field Splitter, JSON Parser, and Stream Selector, include the following logic for records that do not match configured conditions:
  • Discard the record
  • Writes the record to an error log
  • Stop the pipeline
