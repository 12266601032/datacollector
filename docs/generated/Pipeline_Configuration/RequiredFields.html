Required Fields

You can configure required fields for any processor and most target stages. A required field must exist in a record to keep the record in the pipeline. When a record does not include a required field, the stage passes the field to the pipeline for error handling.

Configure required fields based on your business requirements or in order to minimize processing errors.

For example, a Field Hasher encodes social security data in the /SS field. To ensure processing only records that include social security numbers, make /SS a required field.
