Executors

An executor stage executes a task in an external system when it receives an event - it does not write or store events. Use executor stages as part of an event stream to perform event-driven, pipeline-related tasks, such as moving a fully-written file when a destination closes it.

For more information about event handling, see Event Handling.

You can use the following executors in a pipeline:
  • HDFS File Metadata - Changes file metadata upon receiving an event record.
  • Hive Query - Runs a user-defined Hive or Impala query upon receiving an event record.
  • JDBC Query - Runs a user-defined query upon receiving an event record.
  • MapReduce - Starts the specified MapReduce job upon receiving an event record.
