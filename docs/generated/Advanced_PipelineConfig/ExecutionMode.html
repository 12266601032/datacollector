Execution Mode

The Data Collector can run standalone mode or cluster mode. In standalone mode, the Data Collector runs as a single Data Collector process. The Data Collector runs in standalone mode by default.

In cluster mode, the Data Collector runs as an application within Spark, allowing it to process large volumes of data from a Kafka CDH cluster.

Spark is an open source cluster-computing application that is installed as part of the Kafka CDH. Within Spark, the Data Collector spawns a Data Collector worker for each partition in the Kafka cluster. So each partition has a Data Collector worker to process data.

Note: When you add a partition to the Kafka topic, restart the pipeline to enable the Data Collector to generate a new worker to read from the new partition.

Cluster Mode Configuration

To run the Data Collector in cluster mode, perform the following steps:
  1. Install the Data Collector on a Spark on YARN gateway node.
  2. In the Data Collector properties file, set the sdc.execution.mode property to Cluster.
  3. In the pipeline properties, on the General tab, set the Execution Mode property to Cluster.
  4. Use a Kafka Consumer origin in the pipeline.

Monitoring a Cluster

After you start a pipeline, the Data Collector console displays basic monitoring information for the cluster and a list of Data Collector workers in the cluster. To monitor a Data Collector worker, select the worker from the list.
