MapR Prerequisites

Due to licensing restrictions, StreamSets cannot distribute MapR libraries with Data Collector. As a result, you must perform additional steps to enable the Data Collector machine to connect to MapR. You must install the required client libraries and run a MapR setup script to modify configuration files and create the required symbolic links.

Note: The MapR DB destination and the Hive Streaming destination using the MapR library currently only support MapR version 5.1.0. All other stages that use the MapR library support MapR versions 5.0.0 and 5.1.0.
  1. Install Data Collector on a node in the MapR cluster or on a client machine.
    To run Data Collector on a client machine - outside the cluster or on your local machine - the MapR client package must be installed and configured on the machine. If the MapR client package is not installed on the machine, download and install the following files:
    • MapR client library - Typically named mapr-client_<version>.<ext>.
      You can download the files for your operating system here:
      http://package.mapr.com/releases/<version>/
    • Kafka client library - Typically named mapr-kafka-<version>.<ext>.
      You can download the files for your operating system here: 
      http://package.mapr.com/releases/ecosystem-<version>/
  2. From the Data Collector runtime directory, $SDC_DIST, use the following command to make the MapR setup script file executable:
    sudo chmod u+x etc/mapr-setup.sh
  3. To run the setup script, use the following command:
    etc/mapr-setup.sh
  4. When prompted, enter 5.0.0 or 5.1.0 for the MapR version.
  5. When prompted, enter the absolute path to the MapR home directory, usually /opt/mapr.

    The script modifies configuration files and creates the required symbolic links.

  6. When the script finishes, restart Data Collector and verify that you can see the MapR origins and destinations in the stage library.
