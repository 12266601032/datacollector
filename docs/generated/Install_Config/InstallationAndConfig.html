Installation and Configuration

You can install the Data Collector and start it manually or run it as a service.

After installation, you can optionally install additional stages, create additional users, customize the Data Collector, and configure the environment.

If you use Cloudera Manager, you can alternatively install and administer the Data Collector through Cloudera Manager. For more information, see Installation with Cloudera Manager.

Installation Requirements

Install the Data Collector on a machine with the following minimum requirements:

Requirement Description
Operating Systems Use one of the following operating systems and versions:
  • Mac OS X
  • CentOS 6 or 7
  • RedHat Enterprise Linux 6 or 7
  • Ubuntu 14.04
Java Oracle Java 7 or 8
Browser Use the latest version of one of the following browsers:
  • Chrome
  • Firefox
  • Safari

Installation and Launch (Manual Start)

Install the Data Collector on the machine to run pipelines. After installation, you can launch the Data Collector manually.

  1. Download the StreamSets SDC tarball and extract it to the desired location.
  2. Optionally, in the <SDCinstalldir>/libexec/sdc-env.sh file, set the following environment variables:
    • SDC_CONF - The Data Collector configuration directory.
    • SDC_DATA - The Data Collector directory for pipeline state and configuration information.
    • SDC_LOG - The Data Collector directory for Data Collector logs.
    • SDC_RESOURCES - The Data Collector directory for runtime resource files.
    Note: If these variables are not set, the directories are created in the Data Collector installation directory. If this is the case, make sure the user who starts the Data Collector has write permission for the Data Collector installation directory.
  3. Use the following command to launch the Data Collector: 
    $ <SDCinstalldir>/bin/streamsets dc
    Or, use the following command to launch the Data Collector and run it in the background:
    $ nohup <SDCinstalldir>/bin/streamsets dc &

Installation and Launch (Service Start)

Install the Data Collector on the machine to run pipelines. After installation, you can launch the Data Collector.

Installing the Data Collector as a service requires sudo privileges on the root directory.

This procedure walks through setting the default directories, but you can alternatively use the <SDCinstalldir>/libexec/sdcd-env.sh script.

  1. Download the StreamSets SDC tarball and extract it to the desired location, typically /opt/local/.
  2. Copy initd/_sdcinitd_prototype to the /etc/init.d/ directory as sdc.
  3. Edit the sdc file and set the SDC_DIST and SDC_HOME environment variable to the location where you extracted the SDC tarball.
  4. Use the following command to make the sdc file executable: 
    $ sudo chmod 755 /etc/init.d/sdc
  5. Create a system user and group to run the Data Collector.
    The default user and group are sdc/sdc.
  6. Create the /etc/sdc directory and copy all files from etc into this directory.
  7. Create the SDC log directory at /var/log/sdc and change the owner to sdc:sdc.
  8. Create the SDC data directory at /var/lib/sdc and change the owner to sdc:sdc.
  9. Create the SDC data directory at /var/lib/sdc-resources and change the owner to sdc:sdc.
  10. To start the Data Collector as a service, use the following command: 
    $ service sdc start
    For CentOS, use the following command:
    $ chkconfig --add sdc
    For Ubuntu, use the following command:
    $ update-rc.d sdc defaults 97 03
