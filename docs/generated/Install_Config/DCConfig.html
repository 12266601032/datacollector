Data Collector Configuration

You can edit the Data Collector configuration file, sdc.properties, to configure properties such as the host name and port number and account information for email alerts.

You can store sensitive information in separate files and reference them in the Data Collector configuration file. You can also reference information in an environment variable.

You can define runtime properties in the Data Collector configuration file or in a separate file. For more information, see Runtime Properties.

Enabling Kerberos Authentication

You can use Kerberos authentication to connect to origin and destination systems, as well as YARN clusters.

By default, the Data Collector uses the user account who started it to connect to external systems. When you enable Kerberos, it can use the Kerberos principal to connect to external systems.

To enable Kerberos authentication, perform the following steps:
  1. Configure the Data Collector to use Kerberos.

    In the Data Collector configuration file, configure the Kerberos properties to enable Kerberos and define the principal and keytab.

  2. Configure the origin or destination stage to use Kerberos.
    In the pipeline, you can configure Kerberos authentication for the following stages:
    • Hadoop FS origin
    • Hadoop FS destination
    • HBase destination
Note: Cluster pipelines automatically use Kerberos authentication if the YARN cluster requires it, and if Kerberos is enabled for the Data Collector.

Referencing Sensitive Values in Files

For increased security you can store sensitive information, such as a password, in separate files and reference the files in the Data Collector configuration file as follows: @<filename>@.

You can store a single piece of information in a file. When the Data Collector starts, it checks the reference files for information. By default, the Data Collector expects the email password and keystore file password to be stored in files.

For example, if you configure xmail.username as follows, the Data Collector checks the email_username.txt file for the user name:
xmail.username=@email_username.txt@
To store and reference sensitive information:
  1. Create a text file for each configuration value that you want to safeguard.
  2. Include only one configuration value in each file.

    You can store any configuration value in a separate file, but do not include more than one configuration value in a file.

  3. Save the file in the same directory as the Data Collector configuration file: 
    <SDC_installdir>/etc
  4. In the Data Collector configuration file, set the relevant value to the appropriate file name. Surround the file name with @ symbols as follows:
    @<filename>@

HTTP Protocols

You can configure the Data Collector to use HTTP or HTTPS. By default the Data Collector uses HTTP.

HTTPS requires an SSL certificate. The Data Collector provides a self-signed certificate so you can use HTTPS immediately. You can also generate a certificate that is self-signed or signed by a certifying authority. Web browsers generally issue a warning for self-signed certificates.

When you configure the Data Collector to use HTTPS, you can have the Data Collector redirect HTTP requests to HTTPS or to ignore them.

When you generate a SSL certificate, use the following files:
keystore file
A file that contains the private key and self-signed certificates for the web server. You can create your own keystore file or use the keystore file provided by the Data Collector.
Store the keystore file in the following location:
<SDCinstalldir>/etc
The default keystore file is keystore.jks.
keystore password file
A file that contains the password to open the Java keystore file. You can create your own keystore password file or use the file provided by the Data Collector.
Store the keystore password file in the following location:
<SDCinstalldir>/etc
The default keystore file is keystore-password.txt.

Configuring the Data Collector

You can customize the Data Collector by editing the Data Collector configuration file.

Use a text editor to edit the Data Collector configuration file. After you edit the file, restart the Data Collector to enable the changes.

The Data Collector configuration file, sdc.properties, is located in the following location:
<SDCinstalldir>/etc/sdc.properties
The Data Collector configuration file includes the following general properties:
General Property Description
http.port Port number to use for the Data Collector console.

Default is 18630.
https.port Secure port number for the Data Collector console. Any number besides -1 enables the secure port number.

If you use both port properties, the HTTP port bounces to the HTTPS port.

Default is -1.
https.keystore.path Keystore path and file name.

By default, the file name is sdc-keystore.jks and expected in the same directory as the Data Collector configuration file.
https.keystore.password File that contains the password to the keystore file.

By default, the file name is keystore-password.txt and expected in the same directory as the Data Collector configuration file.

When appropriate, you can replace the default file name.
http.authentication HTTP authentication. Use none, basic, digest, or form.

When using basic, digest, or form, use the associated realm.properties file to define user accounts. The realm.properties files are located in <SDCinstall>/etc directory.

Default is form.

http.authentication.login.module Indicates where user account information resides:
  • Set to file to use the realm.properties files.
  • Set to ldap to use an LDAP server.
http.realm.file.permission.check Checks the permissions for the realm.properties file in use:
  • Set to true to ensure that the file allows access only to the owner.
  • Set to false to skip the permission check.

Relevant when http.authentication.login.module is set to file.

http.authentication.ldap.role.mapping Maps groups defined by the LDAP server to Data Collector roles.
Enter a semicolon-separated list as follows:
<ldap group>:<SDC role>,<additional SDC role>,<additional SDC role>);
<ldap group>:<SDC role>,<additional SDC role>...

Relevant when http.authentication.login.module is set to ldap.
kerberos.client.enabled Enables the Data Collector to use Kerberos authentication. Must be enabled to allow a stage to use Kerberos authentication to access external systems.
kerberos.client.principal Kerberos principal for the Data Collector to use. Enter a service principal.
kerberos.client.keytab Location of the Kerberos keytab file that contains the credentials for the Kerberos principal.

Use a fully-qualified directory or a directory relative to the Data Collector configuration directory.
preview.maxBatchSize Maximum number of records used to preview data.

Default is 10.
preview.maxBatches Maximum number of batches used to preview data.

Default is 10.
production.maxBatchSize Maximum number of records included in a batch when the pipeline runs.

Default is 1000.
production.maxErrorRecordsPerStage Maximum number of error records to save in memory for each stage to display in monitor mode. When the limit is reached, older error records are discarded.

Default is 100.

production.maxPipelineErrors Maximum number of pipeline errors to save in memory to display in monitor mode. When the limit is reached, older errors are discarded.

Default is 100.

max.logtail.concurrent.requests Maximum number of external processes allowed to access the Data Collector log file at the same time through REST API calls.

Default is 5.
max.webSockets.current.requests Maximum number of WebSocket calls allowed.
ui.help.base.url Base URL for the local Data Collector online help.

Do not change this value.
ui.hosted.help.base.url Base URL for the Data Collector online help hosted by the StreamSets website.

Do not change this value.
ui.refresh.interval.ms Interval in milliseconds that the Data Collector waits before refreshing the Data Collector console.

Default is 2000.
ui.jvmMetrics.refresh.interval.ms Interval in milliseconds that the JVM metrics are refreshed.

Default is 4000.
ui.enable.usage.data.collection Allows anonymous Google Analytics information to be sent StreamSets.

Default is true.

ui.enable.webSocket Enables the Data Collector to use WebSocket to gather pipeline information.
ui.undo.limit Number of recent actions stored so you can undo them.
The Data Collector configuration file includes the following properties for sending email alerts:
Email Alert Properties Description
mail.transport.protocol Use smtp or smtps.

Default is smtp.
mail.smtp.host SMTP host name.

Default is localhost.
mail.smtp.port SMTP port number.

Default is 25.
mail.smtp.auth Whether the SMTP host uses authentication. Use true or false.

Default is false.
mail.smtp.starttls.enable Whether the SMTP host uses STARTTLS encryption. Use true or false.

Default is false.

mail.smtps.host SMTPS host name.

Default is localhost.
mail.smtps.port SMTPS port number.

Default is 25.
mail.smtps.auth Whether the SMTPS host uses authentication. Use true or false.

Default is false.
xmail.username User name for the email account to send alert email.
xmail.password File that contains the password for the email account.

By default, the file name is email-password.txt and expected in the same directory as the Data Collector configuration file.

When appropriate, you can replace the default file name.
xmail.from.address Email address to send alert email.
The Data Collector configuration file includes the following advanced properties:
Advanced Property Description
runtime.conf.location Location of runtime properties. Use to declare where runtime properties are defined:
  • embedded - The runtime properties are defined in the file.

    Use the following naming convention to define runtime properties in the Data Collector configuration file: runtime.conf_<property name>.

  • <file path> - The directory and file name where runtime properties are defined. The runtime properties file must be local to the Data Collector.

    When you define runtime properties in a separate file, do not prefix the property names with runtime.conf_ .
