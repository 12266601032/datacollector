Full Installation

You can download and install a full version of Data Collector. Use the full installation to install all stage libraries.

To install and launch a full version of Data Collector, download the full installation tarball from the StreamSets website. You can install the full Data Collector and start it manually, or you can start it as a service.

Full Installation and Launch (Manual Start)

To install the full Data Collector and start it manually, download the full Data Collector tarball.

  1. Use the following URL to download the full StreamSets Data Collector tarball from the StreamSets website: https://streamsets.com/opensource.
  2. Extract the tarball to the desired location.
  3. For a production environment, configure the directories used to store configuration, data, log, and resource files so that they are outside of $SDC_DIST, the base Data Collector runtime directory. Use directories outside of the runtime directory to enable use of the directories after Data Collector upgrades.
    For a development or test environment, you can use the default locations within the $SDC_DIST runtime directory. However, StreamSets recommends that you use directories outside of the runtime directory for all environments. If you use the default values for a development or test environment, make sure the user who starts Data Collector has write permission for the base Data Collector runtime directory.
    1. Create directories outside of the $SDC_DIST runtime directory for the configuration, data, log, and resource files.
    2. In the $SDC_DIST/libexec/sdc-env.sh file, set the following environment variables to the newly created directories:
      • SDC_CONF - The Data Collector configuration directory.
      • SDC_DATA - The Data Collector directory for pipeline state and configuration information.
      • SDC_LOG - The Data Collector directory for logs.
      • SDC_RESOURCES - The Data Collector directory for runtime resource files.
    3. Copy all files from $SDC_DIST/etc to the newly created $SDC_CONF directory.
  4. Use the following command to launch Data Collector: 
    $SDC_DIST/bin/streamsets dc
    Or, use the following command to launch Data Collector and run it in the background:
    nohup $SDC_DIST/bin/streamsets dc &
  5. To access the Data Collector console, enter the following URL in the address bar of your browser: 
    http://<system-ip>:18630/

Full Installation and Launch (Service Start)

To install the full Data Collector as a service, download the full Data Collector tarball. Installing Data Collector as a service requires sudo privileges on the root directory.

This procedure walks through setting the default directories and the default system user and group used to start Data Collector as a service. Before you install, you can alternatively use the $SDC_DIST/libexec/sdcd-env.sh file to modify the environment variables that define directories and the system user and group.

  1. Use the following URL to download the Data Collector tarball from the StreamSets website: https://streamsets.com/opensource.
  2. Extract the tarball to the desired location, typically /opt/local/.
  3. Copy initd/_sdcinitd_prototype to the /etc/init.d/ directory as sdc.
  4. Edit the sdc file and set the $SDC_DIST and $SDC_HOME environment variables to the location where you extracted the tarball.
  5. Use the following command to make the sdc file executable: 
    sudo chmod 755 /etc/init.d/sdc
  6. Create a system user and group named sdc.
    The sdc user and group are used to start Data Collector as a service.
  7. Create the Data Collector configuration directory at /etc/sdc and copy all files from etc into this directory.
  8. Change the owner of the /etc/sdc directory and all files in the directory to sdc:sdc.
  9. Use the following command to set owner only permission on the form-realm.properties file in the /etc/sdc directory: 
    chmod go-rwx /etc/sdc/form-realm.properties
  10. Create the Data Collector log directory at /var/log/sdc and change the owner to sdc:sdc.
  11. Create the Data Collector data directory at /var/lib/sdc and change the owner to sdc:sdc.
  12. Create the Data Collector resources directory at /var/lib/sdc-resources and change the owner to sdc:sdc.
  13. To start Data Collector as a service, use the following command: 
    service sdc start
    For CentOS, use the following command:
    chkconfig --add sdc
    For Ubuntu, use the following command:
    update-rc.d sdc defaults 97 03
  14. To access the Data Collector console, enter the following URL in the address bar of your browser: 
    http://<system-ip>:18630/
