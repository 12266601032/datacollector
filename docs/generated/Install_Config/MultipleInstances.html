Multiple Data Collector Instances

You can create multiple Data Collector instances. Each instance uses the same executables, but can be configured differently and run different pipelines.

You might create multiple instances when you want to run several Data Collectors on the same machine. You can run Data Collector instances at the same time as long as sufficient resources are available.

When you create an additional Data Collector instance, the new instance includes any Data Collector customization implemented in the Data Collector configuration file and etc directory, as well as the Data Collector environment configuration file. You can customize the configuration for the new Data Collector instance after creating it.

Creating Multiple Instances

When you create an instance, you specify a directory and http or https port number. The Data Collector creates the directory and copies several directories and configuration files to the directory. It copies files from the Data Collector etc and libexec directories to the new Data Collector instance and updates the http/https port number.
  1. From the command line, if necessary, change to the Data Collector installation directory.
  2. Enter the following command: 
    bin/streamsets create-dc -home=<new SDC directory> (-httpPort=<SDC_HTTP_PORT> | -httpsPort=<SDC_HTTPS_PORT>)
    Note: Enter a directory that does not yet exist.
    The Data Collector creates another Data Collector instance in the specified directory, creating a bin directory and copying the etc and libexec directories to the new instance.
  3. To launch the new Data Collector, navigate to the new Data Collector installation directory and enter the following command: 
    bin/streamsets dc
    The new Data Collector starts running. Use the http host and port number to access the new Data Collector instance.
