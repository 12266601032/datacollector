Users and Roles

The Data Collector includes a default administrator login. You can create additional users for increased security.

The Data Collector provides several roles that define the tasks that a user can perform. When you configure a user, assign the user one or more roles. Users can perform any task that their roles allow.

The Data Collector provides the following roles:
admin
Allows you to perform any task within the Data Collector. As an admin, you can configure pipelines and alerts, start and stop pipelines, view snapshots, and start or stop the Data Collector.
manager
Allows you to manage pipelines. As a manager, you can start and stop pipelines, monitor pipelines, configure and reset alerts, and view snapshots. You cannot configure pipelines or perform data preview.
creator
Allows you to configure pipelines. As a creator, you can configure pipelines and alerts, perform data preview, and monitor the pipeline. You cannot view snapshots or reset alerts.
guest
Allows read-only access to pipelines, alerts, and Data Collector logs. As a guest, you can view pipeline and alert configuration, view general monitoring and log information. You cannot view data preview or snapshots, or configure pipelines or alerts.

Creating Additional Users

The Data Collector includes a default login: admin/admin. You can create additional Data Collector users for increased security.

You can use an md5 program to encrypt login information.

  1. Modify the local-realm.properties file located in the etc directory.
    When installed as a service, the location is /etc/sdc/local-realm.properties.
  2. For each new user, add a line using the following format: 
     <username>: <password>[,<rolename> …]
    Assign one or more roles for each user.
  3. To use a password that is not stored in plain text, use the following format for the password: 
    MD5:<password-hash>
  4. You can generate the hash by running an md5 program, such as md5 on Mac OS X or md5sum on Linux program.
    For example: 
    echo -n "<password>" | md5
  5. To use digest authentication, hash the combination of <username>:<realm>:<password>.
    For example: 
    echo -n "<username>:<realm>:<password>" | md5
  6. To make the new users available, restart the Data Collector.
