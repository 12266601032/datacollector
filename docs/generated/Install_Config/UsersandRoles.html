Users and Roles

The Data Collector provides several roles and a default user account based on each role. For increased security, update the passwords for these accounts or create additional user accounts. A production deployment of the Data Collector should integrate with intranet authentication such as LDAP.

Data Collector roles define the tasks that a user can perform. When you configure a user, assign the user one or more roles. Users can perform any task that their roles allow.

The Data Collector provides the following roles:
admin
Allows you to perform any task within the Data Collector. As an admin, you can configure pipelines and alerts, start and stop pipelines, view snapshots, and start or stop the Data Collector.
The login for the admin user account is admin/admin.
manager
Allows you to manage pipelines. As a manager, you can start and stop pipelines, monitor pipelines, configure and reset alerts, and view snapshots. You cannot configure pipelines or perform data preview.
The login for the manager user account is manager/manager.
creator
Allows you to configure pipelines. As a creator, you can configure pipelines and alerts, perform data preview, and monitor the pipeline. You cannot view snapshots or reset alerts.
The login for the creator user account is creator/creator.
guest
Allows read-only access to areas of the Data Collector. As a guest, you can view pipeline and alert configuration, view general monitoring and log information. You cannot view data preview or snapshots, or configure pipelines or alerts.
The login for the guest user account is guest/guest.

Configuring Users

The Data Collector includes several default users. You can customize the default users or create additional Data Collector users for increased security.

Configure users in the properties file for the type of authentication that you use. For example, if you use basic authentication, you configure users in the basic-realm.properties file.

To encrypt login information, you can use an md5 program such as md5 on Mac OS X or md5sum on Linux.

For Basic and Form authentication, you can encrypt the password. For example: 
echo -n "<password>" | md5
For Digest authentication, you can hash the combination of <username>:<realm>:<password>. For example: 
echo -n "<username>:<realm>:<password>" | md5
  1. Modify the properties file for the type of authentication that you use.
    The file name is <authentication type>-realm.properties.
    The file is located in the following directory: <SDCinstalldir>/etc/. When the Data Collector is installed as a service, the file is located in the follwing directory: /etc/sdc/.
  2. For each new user, add a user definition using the following format: 
    <username>: <password>, user, <role> [, <additional role>, <additional role>...]
    Note: Assign one or more roles for each user. Be sure to include "user" in every user definition.
  3. To use a password that is not stored in plain text, use the following format for the password: 
    MD5:<hashed password>
  4. To make the new users available, restart the Data Collector.
