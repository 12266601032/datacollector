Stream Selector

The Stream Selector passes data to different streams based on user-defined conditions. You can pass data to as many streams as you like. The Stream Selector passes data to all streams where the data matches the condition.

When you configure a condition for a stream, the Stream Selector creates an output node for the stage. When incoming data matches the condition for a stream, the Stream Selector passes the data to the stage connected to the output node. The Stream Selector uses a fallback action for data that does not match the conditions, but you can configure alternative actions.

Handling Non-Matching Records

You can handle data that does not match any configured conditions in one of the following ways:

Use the fallback action
The Fallback Action property determines how the Stream Selector handles records that do not match any conditions. The Stream Selector can perform one of the following fallback actions:
  • Drop the record.
  • Write the record to the error log.
  • Stop the pipeline.
Create a default condition
You can create an additional default condition to capture records that do not match the other conditions. When you create a default condition, you connect the associated node with a stream in the pipeline. You can configure the stream to perform additional processing or write it directly to a destination. If you do not need the records, you can use the Trash destination .

In the following example, Stream 1 passes data with no locale information to a stream that replaces the nulls before writing to the Kafka destination. Stream 2 passes data with the English locale directly to the same Kafka destination. Stream 3 uses the default condition to capture data from other locales and stream it to a different destination.

Configuring the Stream Selector

Use a Stream Selector to route data to different streams.

  1. In the Properties panel, optionally enter a name and description for the stage.
  2. Enter a condition for the first stream.
  3. To create another stream, click the Add icon and then enter the condition for the second stream.
    Repeat this step to configure the number of streams that you need.
  4. To use a default stream to capture any records that do not match the other conditions, click the Add icon and then enter the following condition: ${default}.
  5. For Fallback Action, select the action to take for records that do not match the conditions:
    • Drop record. Drops the record from the pipeline.
    • Record to error. Writes the record to the error log.
    • Fail pipeline. Stops the pipeline.
  6. To define a constant for use in conditions, enter the name and value for the constant.
  7. To add additional constants, click Add and configure the constant.
  8. Optionally include Required Fields for the stage.
    You might want to add any field that you use in condition as a required field.
