Stream Selector

The Stream Selector passes data to streams based on conditions.

When you configure a condition, the Stream Selector creates an output location for the stage that you connect to a downstream stage. The Stream Selector passes a record to all streams where the record matches the condition.

Handling Non-Matching Records

When a record does not match any condition, the Stream Selector can use the configured fallback action or you can configure a default stream that accepts the remaining records:

Use the fallback action
The Fallback Action property determines how the Stream Selector handles records that do not match any conditions. The Stream Selector can perform one of the following fallback actions:
  • Discard the record.
  • Write the record to the error log.
  • Stop the pipeline.
Create a default stream
You can create a default stream to capture records that do not match other conditions. When you create a default stream, you connect the stage output to a branch in the pipeline.
You can configure the branch to perform additional processing or write it directly to a destination. You might write the data to a real destination to be reviewed later, or you might use the Trash destination.
When you use the Trash destination, the pipeline discards the records. But unlike the discard fallback action, you can view the number of records being discarded when you monitor the pipeline, and you can create alerts around this metric.

Configuring the Stream Selector

Use a Stream Selector to route data to different streams.

  1. In the Properties panel, optionally enter a name and description for the stage.
  2. For Required Fields, enter any fields that you want to require data.
    You might include any field that the processor uses.
  3. Enter a condition for the first stream.
  4. To create another stream, click the Add icon and then enter the condition for the second stream.
    Repeat this step to configure the number of streams that you need.
  5. To use a default stream to capture any records that do not match the other conditions, click the Add icon and then enter the following condition: ${default}.
  6. For Fallback Action, select the action to take for records that do not match the conditions:
    • Drop record. Drops the record from the pipeline.
    • Record to error. Writes the record to the error log.
    • Fail pipeline. Stops the pipeline.
  7. To define a constant for use in conditions, enter the name and value for the constant.
  8. To add additional constants, click Add and configure the constant.
