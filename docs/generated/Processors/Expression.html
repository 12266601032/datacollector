Expression Evaluator

The Expression Evaluator performs calculations on incoming data on a row-by-row basis. It writes the results of the calculations to a new field in the pipeline.

To create an expression, you define the name of the field to receive the results of the expression and the expression that you want to use. Use the expression language to define the calculations that you want to use.

You can define constants that you can use in any expression in the processor. Define a constant when you have a constant value that you use in more than one expression or that you want to change easily.

Expression Output Field

When you configure an expression, the Expression Evaluator writes the results of the expression to a field. You can use an existing field or create a new field.

When you use an existing field, the Expression Evaluator replaces the incoming value with the new value. When you use a new field, the Expression Evaluator adds the new field to the record and passes the value of the expression.

Configuring an Expression Evaluator

Configure an Expression Evaluator to perform calculations on records on a row-by-row basis.

  1. In the Properties panel, optionally enter a name and description for the stage.
  2. For Required Fields, enter any fields that you want to require data.
    You might include any field that the processor uses.
  3. To configure an expression, configure the following properties:
    Expression Evaluator Property Description
    Field Name of the field to pass the results of the expression to the next stage. Enter the name of the new or existing field as follows: /<field name>

    The Expression Evaluator replaces values in existing field.

    Expression Expression to evaluate.
  4. Optionally define constants that you want to use within the processor.
    To define a constant, click Add, and then enter a Name and Value for the constant.
