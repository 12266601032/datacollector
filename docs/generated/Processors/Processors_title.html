Processors

  • Processors

    A processor stage represents a type of data processing that you want to perform within the pipeline. You can use as many processors in a pipeline as you need.
  • Expression Evaluator

    The Expression Evaluator performs calculations on incoming data on a row-by-row basis. It writes the results of the calculations to a new field in the pipeline.
  • Deduplicator

    The Deduplicator discards records from the pipeline that include duplicate data. The Deduplicator routes data to two streams, one for unique records and one for duplicate records. Use the Deduplicator to route or discard duplicate data.
  • Field Hasher

    The Field Hasher processor uses an algorithm to encode values in a field. The Field Hasher can use MD5, SHA1 or SHA2 to hash field values.
  • Field Masker

    The Field Masker processor masks string values based on the selected mask type. You can use variable-length, fixed-length, or custom masks.
  • Field Converter

    The Field Converter converts the datatype of a field to a compatible datatype. You might use the Field Converter to convert the data type of a field before performing calculations.
  • Field Remover

    The Field Remover removes fields from the pipeline. Use the Field Remover to discard data that you do not want to use.
  • Field Splitter

    The Field Splitter splits string data in a field based on a separator character and passes the separated data to new fields. Use the Field Splitter to split compound string values into individual components.
  • JSON Parser

    The JSON Parser parses a JSON object embedded in String field and passes the resulting data into the pipeline. Based on the contents of the JSON object, the resulting field is either a string, a map or an array.
  • Stream Selector

    The Stream Selector passes data to streams based on conditions.
  • Value Replacer

    The Value Replacer replaces null values in a field with a constant. The Value Replacer can also replace existing values in a field with null values.
