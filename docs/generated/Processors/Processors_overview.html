Processors

A processor stage represents a type of data processing that you want to perform within the pipeline. You can use as many processors in a pipeline as you need.

The Data Collector provides the following processors:
  • Expression - Performs calculations on a row-by-row basis.
  • Field Converter - Converts the data type of a field.
  • Field Hasher - Uses an algorithm to encode sensitive data.
  • Field Masker - Masks sensitive data.
  • Field Remover. Removes fields from a record.
  • Stream Selector. Routes data to different streams based on conditions.
  • Value Replacer Replaces null values or replaces values with NULLs.
  • Field Splitter.
  • JSON Parser.

Referencing Field Names

Stages display a field name with a preceding slash as follows: /<field name>. For example, a field named CITY displays as /CITY.

Generally, you can select fields from a list. When a list is not available, use a preceding slash when you reference a field name.

Required Fields in Processors

A required field must exist in a record to keep it in the pipeline. When a record does not include a required field, the processor drops the record. You can configure required fields for any processor stage.

To minimize processing errors, define any field that a processor actively uses as a required field.
