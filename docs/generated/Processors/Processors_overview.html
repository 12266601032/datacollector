Processors

A processor stage represents a type of data processing that you want to perform within the pipeline. You can use as many processors in a pipeline as you need.

The Data Collector provides the following processors:
  • Expression Evaluator - Performs calculations on a row-by-row basis.
  • Field Converter - Converts the data type of a field.
  • Field Hasher - Uses an algorithm to encode sensitive string data.
  • Field Masker - Masks sensitive string data.
  • Field Remover - Removes fields from a record.
  • Field Splitter - Splits the string values in a field into different fields.
  • JSON Parser - Parses a JSON object embedded in a string field.
  • Record Deduplicator - Removes duplicate records.
  • Stream Selector - Routes data to different streams based on conditions.
  • Value Replacer - Replaces null values or replaces values with NULLs.

Referencing Field Names

Stages display a field name with a preceding slash as follows: /<field name>. For example, a field named CITY displays as /CITY.

After you preview data or select fields using preview data, you can generally select fields from a list. When a list is not available or when you want to define a new field name, type in the field name with a preceding slash.

For example, when you define an output field for an Expression Evaluator, you can select from a list of fields passed from the previous stage. But to create a new field for the results of the expression, you enter a field name, such as /Code.

Required Fields

You can configure required fields for any processor and most target stages. A required field must exist in a record to keep it in the pipeline. When a record does not include a required field, the stage passes the field to the pipeline for error handling.

Configure required fields based on your business requirements or in order to minimize processing errors.

For example, a Field Hasher encodes social security data in the /SS field. To ensure processing only records that include social security numbers, make /SS a required field.

Selecting Fields with Preview Data

At various points in a pipeline, you can use preview data to help select fields. Preview data displays when a pipeline is configured to a point where you can to perform a data preview.

If preview data for field selection does not display, continue configuring the pipeline and try again.
