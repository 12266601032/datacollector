Processors

A processor stage represents a type of data processing that you want to perform. You can use as many processors in a pipeline as you need.

The Data Collector provides the following processors:
  • Expression Evaluator - Performs calculations on a row-by-row basis.
  • Field Converter - Converts the data type of a field.
  • Field Hasher - Uses an algorithm to encode sensitive string data.
  • Field Masker - Masks sensitive string data.
  • Field Remover - Removes fields from a record.
  • Field Splitter - Splits the string values in a field into different fields.
  • JavaScript Evaluator - Processes records based on custom JavaScript code.
  • JSON Parser - Parses a JSON object embedded in a string field.
  • Jython Evaluator - Processes records based on custom Jython code.
  • Record Deduplicator - Removes duplicate records.
  • Stream Selector - Routes data to different streams based on conditions.
  • Value Replacer - Replaces null values or replaces values with Nulls.

Referencing Field Names

To understand references to fields, think of the record as a root directory. A reference to a field is the path to the field, or field path.

When a record is a line of text, the fields are one level of subdirectories. A reference to a field in a text record is simply: 
/<field name>

So a field named CITY displays as /CITY.

When a record is complex, like a JSON object, the root object might include many additional objects. In this case, a reference to a field is the path to the field: 
/<path to field>/<field name>

For example, a field in a JSON tweet might display as /users/followers_count.

After you preview data or select fields using preview data, you can generally select fields from a list. When a list is not available or when defining a new field name, use the appropriate format to reference the field.

For example, when you define an output field for an Expression Evaluator, you can select from a list of fields passed from the previous stage. But to create a new field for the results of the expression, you enter a field name, such as /Code.

Wildcard Use for Arrays and Maps

In some processor properties / When you list a set of fields in a processor, you can use the asterisk wildcard (*) to represent an index in an array or a key value in a map. Use a wildcard to help define the field path for fields in maps and arrays.

You can use the asterisk wildcard to represent all values as follows:
[*]
Represents all values of an array index. For example, the wildcards in the following field path allow you to mask the social security number of every employee in every division in the company:
Division[*]/Employee[*]/SSN
/*
Represents all values for an element of a map. For example, 
Division[*]/Employee[*]

Selecting Fields with Preview Data

At various points in a pipeline, you can use preview data to help select fields. Preview data displays when a pipeline is configured to a point where you can to perform a data preview.

If preview data for field selection does not display, continue configuring the pipeline and try again.
