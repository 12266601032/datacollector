Jython Evaluator

The Jython Evaluator processes data based on custom Jython code. Use the Jython Evaluator to incorporate custom Jython code in a pipeline. The Jython Evaluator supports Jython version 2.7.

The Jython Evaluator can process data one record at a time or one batch at a time.

When you use a Jython Evaluator in a pipeline, the Data Collector passes a batch of data to the processor and converts the data to an expected format for processing.

Processing Mode

The Jython Evaluator processes data differently based on the processing mode that you select. Use one of the following processing modes:
Record by Record
The processor calls the script for each incoming record. The processor passes the record to the script as a map, and then processes the record.
The code does not require additional error handling logic. The processor handles error records based on the Error Record property defined for the processor.
Use this mode to process one record at a time or to avoid creating error handling logic for the code. Since this mode calls the script for each record, this might affect the performance of the pipeline.
Record Batch
The processor calls the script for each incoming batch of records. The processor passes the batch to the script as an array and processes the batch at one time. The array name is "records."
The script must include error handling code.
Use this mode to process a batch of data at a time. At the end of the script, include the following two functions:
  • out.write (<record>) - Passes the records to the next stage or stages in pipeline.
  • err.write (<record>, <message>) - Passes error records to the pipeline for error handling.

Configuring a Jython Evaluator

Configure a Jython Evaluator to use custom Jython code in a pipeline.

  1. In the Properties panel, on the General tab, configure the following properties:
    General Property Description
    Name Stage name.
    Description Optional description.
    Required Fields Fields that must include data to be passed to the stage.
    Tip: You might include fields that the stage uses.
    On Record Error Error record handling for the stage:
    • Discard - Discards the record.
    • Send to Error - Sends the record to the pipeline for error handling.
    • Stop Pipeline - Stops the pipeline.
  2. On the Jython tab, configure the following properties:
    Jython Property Description
    Record Processing Mode Determines how the Jython Evaluator processes data:
    • Record by Record - Processes records individually. Performs error handling.
    • Record Batch - Processes records in batches. Requires error handling code in the script.
    Script Jython code to use in the processor.
