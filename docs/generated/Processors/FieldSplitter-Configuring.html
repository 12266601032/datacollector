Configuring a Field Splitter

Configure a Field Splitter to split data from a single field into multiple fields. You can split data from a single field. To split additional fields, add another Field Splitter to the pipeline.

  1. In the Properties panel, optionally enter a name and description for the stage.
  2. To split a field, configure the following properties:
    Field Splitter Property Description
    Field to Split The field containing data that you want to split.
    Separator Character to trigger a split.
    Field-Paths for Splits Names of the fields to create for the split data. Precede the name with a slash as follows: /NewField.
    Action for Insufficient Data Action for records that cannot be split to the configured fields:
    • Continue. Passes the record split as much as possible.
    • Discard. Discards the record.
    • Send to Error. Writes the record to the error log.
    • Stop Pipeline. Stops the pipeline.
    Remove Unsplit Value
  3. Optionally, configure Required Fields for the stage.
    You might want to include field that you are splitting as a required field.
