Field Hasher

The Field Hasher processor uses an algorithm to encode values in a field. The Field Hasher can use MD5, SHA1 or SHA2 to hash field values.

Use the Field Hasher to encode highly-sensitive data or data that you might want to decode at a later date. For example, you might use the Field Hasher to encode social security or credit card numbers.

Tip: To hash non-string data, you might use the Field Converter to convert non-string data to String, then pass the data to the Field Hasher.

Configuring a Field Hasher

Configure a Field Hasher to encode sensitive data.

When you configure the Field Hasher, you select the field that you want to encode and the algorithm that you want to use.
  1. In the Properties panel, optionally enter a name and description for the stage.
  2. For Required Fields, enter any fields that you want to require data.
    You might include any field that the processor uses.
  3. On the Hash tab, configure the following properties:
    Field Hasher Property Description
    Fields to Hash One or more String fields to hash with the same algorithm.
    Hash Type Algorithm to use to hash field values:
    • MD5 - Produces a 128-bit (16-byte) hash value, typically expressed in text format as a 32 digit hexadecimal number.
    • SHA1 - Produces a 160-bit (20-byte) hash value.
    • SHA2 - Based on SHA1, but uses a set of four hash functions: 224, 256, 384, or 512 bits.
  4. To hash another field, click the Add icon and then repeat the previous step.
