Field Masker

The Field Masker processor masks string values based on the selected mask type. You can use variable-length, fixed-length, or custom masks.

Use the Field Masker to mask sensitive string data. For example, you might use the Field Masker to mask social security numbers.

Tip: To mask non-string data, you might use the Field Type Converter processor to convert non-string data to String, then stream the data to the Field Masker.

Mask Types

You can use the following mask types to mask data:

Fixed-length
Replaces values with a fixed-length mask. Use when you want to mask any variability in length in the data.
The following example uses a fixed-length masks to hide passwords:

Custom
Replaces values with a mask based on a user-defined pattern. When you define the pattern for the mask, you can use the following special characters:
  • Hash mark (#) - Reveals the original character in that location.
All other characters are used as constants in the mask. For example, the following mask pattern reveals the area code of a phone number: ###-xxx-xxxx.

Configuring a Field Masker

Configure a Field Masker to mask sensitive data.

  1. In the Properties panel, optionally enter a name and description for the stage.
  2. To mask a field, configure the following properties:
    Field Masker Configuration Property Description
    Mask Field String field to be masked.
    Mask Type Mask type to hide field values. Select one of the following options:
    • Fixed-length - Replaces values with a fixed-length mask.
    • Variable-length - Replaces values with a mask the length of the original value.
    • Custom - Replaces values with a user-defined mask.
    Custom Pattern Mask pattern for a custom mask. Enter the pattern that you want to use.

    You can use the hash mark (#) to display characters in the specified location. For example, ###xxx reveals the first three characters of a string.
  3. To mask another field, click Add, and then repeat the previous step.
  4. Optionally include Required Fields for the stage.
    You might want to add the fields that you are masking as required fields.
