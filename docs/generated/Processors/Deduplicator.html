Deduplicator

The Deduplicator discards records from the pipeline that include duplicate data. The Deduplicator routes data to two streams, one for unique records and one for duplicate records. Use the Deduplicator to route or discard duplicate data.

The Deduplicator can compare entire records or a subset of fields. Use a subset of fields to focus the comparison of duplicate data on the fields of particular concern. For example, to discard duplicate purchase events, you might compare information about the purchaser, selected items, and shipping address, but ignore the timestamp of the event.

To enhance pipeline performance, the Deduplicator hashes the comparison fields and uses the hashed values when evaluating for duplicates.

Comparison Window

The Deduplicator compares records up to a user-defined limit. It can also perform comparisons within a window of time.

The Deduplicator compares records until it reaches the record limit. Then, it discards the information kept in memory and starts fresh. You can configure a time limit to trigger a refresh at regular time intervals.

When you stop the pipeline, the Deduplicator discards all information in memory.

Configuring a Deduplicator

Use a Deduplicator to remove records with duplicate data.

  1. In the Properties panel, optionally enter a name and description for the stage.
  2. For Required Fields, enter any fields that you want to require data.
    You might include any field that the processor uses.
  3. On the Deduplication tab, configure the following properties:
    Deduplication Property Description
    Max Records to Compare The maximum number of records to compare.
    Time to Compare Amount of seconds to compare records. This property takes precedence over the number of records.
