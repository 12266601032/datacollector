Starting a Pipeline

You can start a pipeline when it is valid. When you start a pipeline, the Data Collector runs the pipeline until you stop the pipeline or shut down the Data Collector.

For a Directory origin, when you restart a pipeline, the Data Collector starts the pipeline from where it last stopped by default. You can reset the origin to read all available data.

A Kafka Consumer origin starts processing data based on the offset passed from the Kafka ZooKeeper.

A File Tail origin processes data that is written to the file after you start the pipeline.

  1. Verify that the pipeline that you want to use appears in the pipeline canvas.
    If it does not, select the pipeline from the Pipeline library.
  2. To start the pipeline, click the Start icon.
    If the Start icon is not enabled, the pipeline is not valid or the Data Collector is running a different pipeline.

Resetting the Origin

You can reset the origin when you want the Data Collector to process all available data. You can reset the origin for Directory origins. Reset the origin when the pipeline is not running.

For the Directory origin, the Data Collector notes where it stopped processing data when you stop a pipeline. When you restart the pipeline, it continues from where it left off by default.

When you want the Data Collector to process all available data instead of continuing from where it stopped, reset the origin.

To change the starting point for a Kafka Consumer, you need to change the offset in the Kafka ZooKeeper. For details, see the Kafka documentation.

  1. Verify that the pipeline that you want to use appears in the pipeline canvas.
    If it does not, select the pipeline from the Pipeline library.
  2. To reset the origin, click the More icon.
  3. Click Reset Origin.
  4. In the Reset Origin Confirmation dialog box, click Yes to reset the origin.
