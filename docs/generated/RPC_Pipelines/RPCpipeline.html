RPC Pipelines

Remote Protocol Call pipelines, a.k.a. RPC pipelines, are pipelines that pass data from one pipeline to another over the WAN without writing to an intermediary system. Use RPC pipelines to securely transmit data across data centers.

Typically, a pipeline uses a standard origin such as Directory or Kafka Consumer and writes to a standard destination such as HBase. An RPC pipeline writes directly to another RPC pipeline using the RPC destination and RPC origin as follows:

This allows you to process data from one data center and pass it from one data center securely. Effectively, it's like creating a single pipeline that happens to span two data centers.

A set of RPC pipelines process data like a single pipeline: The origin pipeline creates a batch, passes it through the pipeline, then passes it to the destination pipeline. Only when the destination pipeline writes the data to its destination system does the origin pipeline commit the offset.
