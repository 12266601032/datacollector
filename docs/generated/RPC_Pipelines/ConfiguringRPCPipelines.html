Configuration Guidelines for RPC Pipelines

To create a valid set of RPC pipelines, some configuration options must be aligned. Use the following guidelines to configure RPC pipelines:

RPC origin pipeline
In the origin pipeline, configure the following elements:
  1. In the pipeline properties, set the Delivery Guarantee to the option you want to use.
  2. In the pipeline, use the RPC destination.
  3. When you configure the RPC destination, enter the RPC ID, the RPC connection information, and the optional properties as needed.
RPC destination pipeline
In the destination pipeline, configure the following elements:
  1. In the pipeline properties, set the Delivery Guarantee to the same option as the origin pipeline. Using a different option causes the Data Collector to default to At Least Once.
  2. In the pipeline, use the RPC origin.
  3. When you configure the RPC origin, define the properties as follows:
    • Use the same RPC ID used in the RPC destination
    • For the RPC Listening Port, use one of the port numbers defined in the RPC destination RPC connections.
    • Define optional properties as needed.
